# 数据库表结构（用户 / 角色 / 权限 / 菜单）

> 依据现有前端字段与交互推导，采用 MySQL 8.0 设计；字段命名尽量与前端保持一致或提供清晰映射。主数据表为 `users`、`roles`、`permissions`、`menus`，并通过关联表实现多对多关系。

## 设计说明
- 对齐前端字段：如 `userName`、`userEmail`、`roleId`、`roleName`、`status`、菜单的 `isMenu`、`isEnable`、`keepAlive`、`isHide` 等。
- 权限模型：角色-权限多对多；支持按钮级(`BUTTON`)、页面级(`PAGE`)与接口级(`API`)权限。
- 菜单为树形结构：`parent_id` 自引用，支持外链、iframe、缓存、隐藏、徽章、固定标签、全屏等属性。
- 标准审计字段：`created_at`、`updated_at`、`deleted_at`；统一 `InnoDB`、`utf8mb4`。

## 用户表（users）
| 字段名 | 类型 | 是否空 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | NO | AUTO_INCREMENT | 主键 |
| account | VARCHAR(64) | NO |  | 登录账号（唯一） |
| userName | VARCHAR(64) | NO |  | 显示昵称 |
| userEmail | VARCHAR(128) | YES |  | 邮箱（可唯一） |
| userPhone | VARCHAR(32) | YES |  | 手机号（可唯一） |
| avatar | VARCHAR(255) | YES |  | 头像 URL |
| status | TINYINT(1) | NO | 1 | 启用状态：0 禁用，1 启用 |
| password_hash | VARCHAR(255) | NO |  | 密码哈希 |
| salt | VARCHAR(32) | YES |  | 密码盐（可选） |
| last_login_at | DATETIME | YES |  | 最近登录时间 |
| created_at | DATETIME | NO | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |
| deleted_at | DATETIME | YES |  | 软删除时间 |

- 索引与约束
  - 唯一索引：`uniq_users_account (account)`
  - 可选唯一索引：`uniq_users_email (userEmail)`、`uniq_users_phone (userPhone)`
  - 常规索引：`idx_users_status (status)`、`idx_users_last_login (last_login_at)`

## 角色表（roles）
| 字段名 | 类型 | 是否空 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | NO | AUTO_INCREMENT | 主键 |
| roleId | VARCHAR(32) | NO |  | 角色编码（如 `R_SUPER`/`R_ADMIN`/`R_USER`），唯一 |
| roleName | VARCHAR(64) | NO |  | 角色名称 |
| enabled | TINYINT(1) | NO | 1 | 启用状态（0/1） |
| sort | INT | NO | 100 | 排序（小值优先） |
| remarks | VARCHAR(255) | YES |  | 备注 |
| created_at | DATETIME | NO | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |
| deleted_at | DATETIME | YES |  | 软删除时间 |

- 索引与约束
  - 唯一索引：`uniq_roles_roleId (roleId)`
  - 常规索引：`idx_roles_enabled (enabled)`、`idx_roles_sort (sort)`

## 权限表（permissions）
| 字段名 | 类型 | 是否空 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | NO | AUTO_INCREMENT | 主键 |
| code | VARCHAR(64) | NO |  | 权限编码（唯一；映射 `v-auth`/`hasAuth('...')`；对应 `authName`） |
| name | VARCHAR(64) | NO |  | 权限名称（对应 `authLabel`） |
| icon | VARCHAR(64) | YES |  | 图标（对应 `authIcon`） |
| sort | INT | NO | 100 | 排序（对应 `authSort`） |
| type | ENUM('PAGE','BUTTON','API') | NO | 'BUTTON' | 权限类型 |
| menu_id | BIGINT UNSIGNED | YES |  | 所属菜单（可选；按钮/接口权限挂载页面） |
| enabled | TINYINT(1) | NO | 1 | 启用状态 |
| created_at | DATETIME | NO | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |
| deleted_at | DATETIME | YES |  | 软删除时间 |

- 索引与约束
  - 唯一索引：`uniq_permissions_code (code)`
  - 外键：`fk_permissions_menu (menu_id) REFERENCES menus(id) ON DELETE SET NULL`
  - 常规索引：`idx_permissions_type (type)`、`idx_permissions_enabled (enabled)`、`idx_permissions_sort (sort)`

| is_menu | TINYINT(1) | NO | 1 | 是否菜单（0=非菜单，仅权限节点；1=菜单） |

| sort | INT | NO | 100 | 排序 |
| keep_alive | TINYINT(1) | NO | 0 | 是否缓存 |
| is_hide | TINYINT(1) | NO | 0 | 是否隐藏 |
| link_url | VARCHAR(255) | YES |  | 外链地址 |
| is_iframe | TINYINT(1) | NO | 0 | 是否 iframe |
| badge_count | INT | YES |  | 数字徽章 |
| badge_text | VARCHAR(32) | YES |  | 文本徽章 |
| fixed_tab | TINYINT(1) | NO | 0 | 固定标签 |
| active_path | VARCHAR(128) | YES |  | 激活路径 |
| is_full_page | TINYINT(1) | NO | 0 | 全屏页面 |
| created_at | DATETIME | NO | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |
| deleted_at | DATETIME | YES |  | 软删除时间 |

- 索引与约束
  - 唯一索引：`uniq_menus_name (name)`、`uniq_menus_path (path)`
  - 外键：`fk_menus_parent (parent_id) REFERENCES menus(id) ON DELETE CASCADE`
  - 常规索引：`idx_menus_parent_sort (parent_id, sort)`、`idx_menus_enable (is_enable)`、`idx_menus_is_menu (is_menu)`

## 关联表（关系实现）

### 用户-角色（user_roles）
| 字段名 | 类型 | 是否空 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | NO | AUTO_INCREMENT | 主键 |
| user_id | BIGINT UNSIGNED | NO |  | 用户 ID |
| role_id | BIGINT UNSIGNED | NO |  | 角色 ID |
| created_at | DATETIME | NO | CURRENT_TIMESTAMP | 创建时间 |

- 索引与约束：
  - 唯一约束：`uniq_user_roles (user_id, role_id)`
  - 外键：`fk_ur_user (user_id) REFERENCES users(id) ON DELETE CASCADE`、`fk_ur_role (role_id) REFERENCES roles(id) ON DELETE CASCADE`

### 角色-权限（role_permissions）
| 字段名 | 类型 | 是否空 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | NO | AUTO_INCREMENT | 主键 |
| role_id | BIGINT UNSIGNED | NO |  | 角色 ID |
| permission_id | BIGINT UNSIGNED | NO |  | 权限 ID |
| created_at | DATETIME | NO | CURRENT_TIMESTAMP | 创建时间 |

- 索引与约束：
  - 唯一约束：`uniq_role_permissions (role_id, permission_id)`
  - 外键：`fk_rp_role (role_id) REFERENCES roles(id) ON DELETE CASCADE`、`fk_rp_permission (permission_id) REFERENCES permissions(id) ON DELETE CASCADE`

### 角色-菜单（role_menus）
| 字段名 | 类型 | 是否空 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | NO | AUTO_INCREMENT | 主键 |
| role_id | BIGINT UNSIGNED | NO |  | 角色 ID |
| menu_id | BIGINT UNSIGNED | NO |  | 菜单 ID |
| created_at | DATETIME | NO | CURRENT_TIMESTAMP | 创建时间 |

- 索引与约束：
  - 唯一约束：`uniq_role_menus (role_id, menu_id)`
  - 外键：`fk_rm_role (role_id) REFERENCES roles(id) ON DELETE CASCADE`、`fk_rm_menu (menu_id) REFERENCES menus(id) ON DELETE CASCADE`

> 注：也可仅通过 `role_permissions` 控制菜单访问（当 `permissions.type='PAGE'` 与菜单关联时），`role_menus` 作为显隐和导航控制的补充。

## 示例建表 SQL（MySQL 8.0）

```sql
-- users
CREATE TABLE IF NOT EXISTS users (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  account VARCHAR(64) NOT NULL,
  userName VARCHAR(64) NOT NULL,
  userEmail VARCHAR(128) NULL,
  userPhone VARCHAR(32) NULL,
  avatar VARCHAR(255) NULL,
  status TINYINT(1) NOT NULL DEFAULT 1,
  password_hash VARCHAR(255) NOT NULL,
  salt VARCHAR(32) NULL,
  last_login_at DATETIME NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at DATETIME NULL,
  PRIMARY KEY (id),
  UNIQUE KEY uniq_users_account (account),
  UNIQUE KEY uniq_users_email (userEmail),
  UNIQUE KEY uniq_users_phone (userPhone),
  KEY idx_users_status (status),
  KEY idx_users_last_login (last_login_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- roles
CREATE TABLE IF NOT EXISTS roles (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  roleId VARCHAR(32) NOT NULL,
  roleName VARCHAR(64) NOT NULL,
  enabled TINYINT(1) NOT NULL DEFAULT 1,
  sort INT NOT NULL DEFAULT 100,
  remarks VARCHAR(255) NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at DATETIME NULL,
  PRIMARY KEY (id),
  UNIQUE KEY uniq_roles_roleId (roleId),
  KEY idx_roles_enabled (enabled),
  KEY idx_roles_sort (sort)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- menus
CREATE TABLE IF NOT EXISTS menus (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  parent_id BIGINT UNSIGNED NULL,
  name VARCHAR(64) NOT NULL,
  label VARCHAR(128) NOT NULL,
  path VARCHAR(128) NOT NULL,
  component VARCHAR(128) NULL,
  icon VARCHAR(64) NULL,
  is_menu TINYINT(1) NOT NULL DEFAULT 1,
  is_enable TINYINT(1) NOT NULL DEFAULT 1,
  sort INT NOT NULL DEFAULT 100,
  keep_alive TINYINT(1) NOT NULL DEFAULT 0,
  is_hide TINYINT(1) NOT NULL DEFAULT 0,
  link_url VARCHAR(255) NULL,
  is_iframe TINYINT(1) NOT NULL DEFAULT 0,
  badge_count INT NULL,
  badge_text VARCHAR(32) NULL,
  fixed_tab TINYINT(1) NOT NULL DEFAULT 0,
  active_path VARCHAR(128) NULL,
  is_full_page TINYINT(1) NOT NULL DEFAULT 0,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at DATETIME NULL,
  PRIMARY KEY (id),
  UNIQUE KEY uniq_menus_name (name),
  UNIQUE KEY uniq_menus_path (path),
  KEY idx_menus_parent_sort (parent_id, sort),
  KEY idx_menus_enable (is_enable),
  KEY idx_menus_is_menu (is_menu),
  CONSTRAINT fk_menus_parent FOREIGN KEY (parent_id) REFERENCES menus(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- permissions
CREATE TABLE IF NOT EXISTS permissions (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  code VARCHAR(64) NOT NULL,
  name VARCHAR(64) NOT NULL,
  icon VARCHAR(64) NULL,
  sort INT NOT NULL DEFAULT 100,
  type ENUM('PAGE','BUTTON','API') NOT NULL DEFAULT 'BUTTON',
  menu_id BIGINT UNSIGNED NULL,
  enabled TINYINT(1) NOT NULL DEFAULT 1,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at DATETIME NULL,
  PRIMARY KEY (id),
  UNIQUE KEY uniq_permissions_code (code),
  KEY idx_permissions_type (type),
  KEY idx_permissions_enabled (enabled),
  KEY idx_permissions_sort (sort),
  CONSTRAINT fk_permissions_menu FOREIGN KEY (menu_id) REFERENCES menus(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- user_roles
CREATE TABLE IF NOT EXISTS user_roles (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  user_id BIGINT UNSIGNED NOT NULL,
  role_id BIGINT UNSIGNED NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  UNIQUE KEY uniq_user_roles (user_id, role_id),
  KEY idx_ur_user (user_id),
  KEY idx_ur_role (role_id),
  CONSTRAINT fk_ur_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  CONSTRAINT fk_ur_role FOREIGN KEY (role_id) REFERENCES roles(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- role_permissions
CREATE TABLE IF NOT EXISTS role_permissions (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  role_id BIGINT UNSIGNED NOT NULL,
  permission_id BIGINT UNSIGNED NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  UNIQUE KEY uniq_role_permissions (role_id, permission_id),
  KEY idx_rp_role (role_id),
  KEY idx_rp_permission (permission_id),
  CONSTRAINT fk_rp_role FOREIGN KEY (role_id) REFERENCES roles(id) ON DELETE CASCADE,
  CONSTRAINT fk_rp_permission FOREIGN KEY (permission_id) REFERENCES permissions(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- role_menus
CREATE TABLE IF NOT EXISTS role_menus (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  role_id BIGINT UNSIGNED NOT NULL,
  menu_id BIGINT UNSIGNED NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  UNIQUE KEY uniq_role_menus (role_id, menu_id),
  KEY idx_rm_role (role_id),
  KEY idx_rm_menu (menu_id),
  CONSTRAINT fk_rm_role FOREIGN KEY (role_id) REFERENCES roles(id) ON DELETE CASCADE,
  CONSTRAINT fk_rm_menu FOREIGN KEY (menu_id) REFERENCES menus(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 字段映射与前端对应（速览）
- 用户：`account`（登录）、`userName`（昵称）、`userEmail`、`userPhone`、`avatar`、`status`。
- 角色：`roleId`（编码用于 `v-roles`）、`roleName`、`enabled`、`sort`。
- 权限：`code`（用于 `v-auth`/`hasAuth()`）、`name`、`icon`、`sort`、`type`、`menu_id`、`enabled`。
- 菜单：`name`、`label`、`path`、`component`、`icon`、`is_menu`、`is_enable`、`sort`、`keep_alive`、`is_hide`、`link_url`、`is_iframe`、`badge_count`、`badge_text`、`fixed_tab`、`active_path`、`is_full_page`、`parent_id`。

## 使用建议
- 返回给前端的用户信息附带 `roles` 与扁平 `permissions` 列表，菜单通过角色过滤生成可见树。
- 统一采用软删除，业务层过滤 `deleted_at IS NULL`；启用/禁用使用 `status`/`enabled` 控制。
- 结合前端：`role.roleName` 显示、`currentUser.roles` 数组、`v-auth`/`v-roles` 与路由 `meta.roles`、`menuId` 授权按钮在页面下显示。
