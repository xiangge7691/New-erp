# FileStorageConfig 子类型全局统一

## 核心规则
- 父类型定义顶级目录：`put("PERSONNEL", "人员管理")`
- 子类型全局统一：`put("CERTIFICATE", "证书")`、`put("ATTACHMENT", "附件")`
- 前端传 `PERSONNEL_CERTIFICATE` → 自动拼接为 `人员管理/证书`

## 需要修改的内容

### 1. 简化 subTypeDirMap
移除分组注释，所有子类型平铺，全局唯一映射

### 2. 统一所有业务类型命名
将不规范的 businessType 统一为 `{PARENT}_{CHILD}` 格式：

| 旧名称 | 新名称 | 目录路径 |
|---|---|---|
| DISINFECTION_RECORD | ROOM_DISINFECTION | 车间环境/消毒记录 |
| CLEAN_INSPECTION_RECORD | ROOM_CLEAN_INSPECTION | 车间环境/洁净检测 |
| TEMPERATURE_HUMIDITY_RECORD | ROOM_TEMPERATURE_HUMIDITY | 车间环境/温湿度记录 |
| PRESSURE_DIFFERENCE_RECORD | ROOM_PRESSURE_DIFFERENCE | 车间环境/压差记录 |
| STOCK_IN_PURCHASE 等 | 保持不变（已符合规范） | |

### 3. 人员档案只保留两个子类型
- `PERSONNEL_CERTIFICATE` → `人员管理/证书`
- `PERSONNEL_ATTACHMENT` → `人员管理/附件`

### 涉及文件
1. `Common/Config/FileStorageConfig.java` — 简化 subTypeDirMap
2. `Controller/FileController.java` — 更新注释
3. 所有使用旧 businessType 名称的 Controller/Service — 统一改为新名称

### 验证
- `./mvnw compile` 编译通过
