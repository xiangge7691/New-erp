# ERP项目代码质量提升实施计划

## 变更概要

4项变更，按依赖顺序执行：
1. **全局异常处理器** — 创建 `@RestControllerAdvice`，移除 Controller 中 209 个 try-catch
2. **MyBatis-Plus 审计字段自动填充** — 创建基类 + `MetaObjectHandler`，移除 107+ 处手动赋值
3. **MapStruct 替换手动 DTO 转换** — 添加依赖，创建 Mapper 接口，移除所有手写转换
4. **ApiResponse 增强** — 添加错误码常量

安全配置不动。

## 变更2：MyBatis-Plus 审计字段自动填充

### 2.1 新建文件
- **`Data/Entity/AuditEntity.java`** — 审计字段基类
  ```java
  @Data
  public abstract class AuditEntity {
      @TableField(fill = FieldFill.INSERT)
      private Long createdBy;
      @TableField(fill = FieldFill.INSERT_UPDATE)
      private Long updatedBy;
      @TableField(fill = FieldFill.INSERT)
      private LocalDateTime createdTime;
      @TableField(fill = FieldFill.INSERT_UPDATE)
      private LocalDateTime updatedTime;
  }
  ```

- **`Common/Config/MybatisPlusMetaObjectHandler.java`**
  - `@Component` 实现 `MetaObjectHandler`
  - `insertFill`: 设置 `createdTime` = now, `updatedTime` = now, `createdBy` = EntityUtils.getCurrentUserId(), `updatedBy` = EntityUtils.getCurrentUserId()
  - `updateFill`: 设置 `updatedTime` = now, `updatedBy` = EntityUtils.getCurrentUserId()

## 变更3：MapStruct 替换手动 DTO 转换

### 3.1 修改 pom.xml — 添加 MapStruct 依赖
```xml
<dependency>
    <groupId>org.mapstruct</groupId>
    <artifactId>mapstruct</artifactId>
    <version>1.6.3</version>
</dependency>
<dependency>
    <groupId>org.mapstruct</groupId>
    <artifactId>mapstruct-processor</artifactId>
    <version>1.6.3</version>
    <scope>provided</scope>
</dependency>
```
同时在 `maven-compiler-plugin` 中配置 `annotationProcessorPaths`。

## 风险评估

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 字段名统一导致数据库查询失败 | 高 | 提供完整 SQL 脚本，先执行 SQL 再启动 |
| MapStruct 编译时未生成实现类 | 高 | 确保 annotationProcessorPaths 配置正确 |
| 移除 try-catch 后异常信息丢失 | 中 | GlobalExceptionHandler 保留异常消息 |
| 移除审计字段赋值后字段为 null | 中 | MetaObjectHandler 测试覆盖所有实体 |
| 部分 Controller 的 catch 块有特殊逻辑 | 低 | 逐个检查，特殊逻辑迁移到 Service 层 |
