# 统一文件管理：通过 businessType + businessId 关联

## 核心思路
- 所有业务单据不再存储 file_id，统一通过 `file_info` 表的 `business_type` + `business_id` 查询关联文件
- `file_info` 新增 `custom_path` 字段，前端可传自定义目录路径

---

## 一、数据库变更

### 1. file_info 表新增 custom_path 字段
```sql
ALTER TABLE file_info ADD COLUMN custom_path VARCHAR(500) DEFAULT NULL COMMENT '自定义存储子目录';
```

### 2. personnel_certificate 表删除 file_id
```sql
ALTER TABLE personnel_certificate DROP COLUMN file_id;
```

### 3. personnel_file 表删除单文件字段
```sql
ALTER TABLE personnel_file DROP COLUMN health_file;
ALTER TABLE personnel_file DROP COLUMN attachments;
```

---

## 二、后端变更

### 1. FileInfo Entity
- 新增 `customPath` 字段

### 2. FileStorageServiceImpl
- `uploadFileWithBusinessPath()` 增加 `customPath` 参数
- 路径构造：`{basePath}/{businessTypeDir}/{customPath}/{yyyy}/{MM}/{entityName}/{uuid.ext}`
- 不传 customPath 时保持原有逻辑

### 3. FileInfoServiceImpl
- `uploadFileWithBusinessPath()` 增加 `customPath` 参数透传

### 4. FileController
- `/api/files/upload-business` 增加可选参数 `customPath`

### 5. FileStorageConfig
- 添加业务类型映射：
  - `PERSONNEL_CERTIFICATE` → `"人员档案/证书"`
  - `PERSONNEL_HEALTH_FILE` → `"人员档案/健康档案"`
  - `PERSONNEL_ATTACHMENT` → `"人员档案/附件"`

### 6. PersonnelCertificate Entity
- 删除 `fileId` 字段

### 7. PersonnelFile Entity
- 删除 `healthFile`、`attachments` 字段

### 8. PersonnelCertificateController
新增端点：
```
POST   /{id}/attachments          — 上传文件到证书
GET    /{id}/attachments          — 获取证书的所有文件
DELETE /{id}/attachments/{fileId} — 删除证书的某个文件
```

### 9. PersonnelFileController
新增端点：
```
POST   /{id}/health-files          — 上传健康档案文件
GET    /{id}/health-files          — 获取健康档案文件列表
DELETE /{id}/health-files/{fileId} — 删除健康档案文件
POST   /{id}/attachments           — 上传附件文件
GET    /{id}/attachments           — 获取附件文件列表
DELETE /{id}/attachments/{fileId}  — 删除附件文件
```

### 10. Service 层清理
- 删除所有设置 `fileId`/`healthFile`/`attachments` 的代码
- 删除业务记录时同步删除 file_info

### 11. XML Mapper 清理
- PersonnelCertificateMapper.xml：删除 fileId 映射
- PersonnelFileMapper.xml：删除 healthFile/attachments 映射
