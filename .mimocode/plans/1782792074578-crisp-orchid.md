# Plan: 首页仪表盘接口扩展

## 首页需要的数据（来自HTML分析）

### 1. 核心指标卡片（5个）
- **预估产值**: 订单列表 批次数量×结算单价 求和
- **总订单量**: 订单列表 创建时间在筛选范围内
- **总交付量**: 订单列表 有交付时间的
- **总采购额**: 入库信息 金额求和
- **待生产数量**: 生产详情 草稿+已确认 总和

### 2. 待办事项
- **设备维保**: 维保到期提醒（已有 EquipmentMaintenanceService.findUpcomingMaintenance）
- **库存预警**: 库存有效期到期提醒（已有 StockWarningStatsDto）
- **环境管理**: 许可证到期提醒（预留）
- **人员管理**: 健康证到期提醒（已有 PersonnelFileService.findExpiringHealthCerts）

### 3. 订单跟踪看板
- 订单列表 + 生产计划的进度节点（下单→采购→生产→检验→出库→归档）

### 4. 统计图数据
- 交付数量按剂型（月度）
- 预估产值（月度）
- 库存资金占用

## 修改文件清单

### 1. 新增 DTO 类
**目录**: `src/main/java/com/tonghui/erp/Common/Dto/Dashboard/`

- `DashboardMetricsDto.java` — 5个核心指标
- `TodoItemDto.java` — 待办事项项
- `TodoListDto.java` — 待办事项列表（含分类计数）
- `OrderTrackingDto.java` — 订单跟踪项
- `ChartDataDto.java` — 图表数据

### 2. 扩展 DashboardController
**文件**: `src/main/java/com/tonghui/erp/Controller/DashboardController.java`

新增端点：
```
GET /api/dashboard/metrics?startMonth=&endMonth=  — 核心指标
GET /api/dashboard/todos                          — 待办事项列表
GET /api/dashboard/order-tracking?startMonth=&endMonth=&status=  — 订单跟踪
GET /api/dashboard/charts?startMonth=&endMonth=   — 图表数据
```

### 3. 注入更多 Service
- EquipmentMaintenanceService（维保提醒）
- PersonnelFileService（健康证提醒）
- PurchaseOrdersService（采购订单）
- StockInService（入库金额）
