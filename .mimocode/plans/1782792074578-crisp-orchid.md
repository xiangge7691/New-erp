# Plan: 生产管理详情 - 添加6个时间字段 + 时间筛选

## 背景
生产计划主表 `production_plan` 需要新增6个业务时间字段（生产开始、生产结束、检验开始、检验结束、出库时间、归档时间），前端手动填写，并支持对这些时间字段进行独立的时间范围筛选。

## 修改文件清单

### 1. Entity: `ProductionPlan.java`
**文件**: `src/main/java/com/tonghui/erp/Data/Entity/ProductionPlan.java`

在现有字段后（`totalAmount` 之后）添加6个 `LocalDateTime` 字段：

```java
@TableField(value = "production_start_time")
private LocalDateTime productionStartTime;  // 生产开始时间

@TableField(value = "production_end_time")
private LocalDateTime productionEndTime;    // 生产结束时间

@TableField(value = "inspection_start_time")
private LocalDateTime inspectionStartTime;  // 检验开始时间

@TableField(value = "inspection_end_time")
private LocalDateTime inspectionEndTime;    // 检验结束时间

@TableField(value = "outbound_time")
private LocalDateTime outboundTime;         // 出库时间

@TableField(value = "archive_time")
private LocalDateTime archiveTime;          // 归档时间
```

### 2. Mapper XML: `ProductionPlanMapper.xml`
**文件**: `src/main/resources/Data/mapper/ProductionPlanMapper.xml`

在 `BaseResultMap` 和 `Base_Column_List` 中添加6个字段映射。

### 3. Service 接口: `ProductionPlanService.java`
**文件**: `src/main/java/com/tonghui/erp/Service/ProductionPlanService.java`

修改 `queryProductionPlans` 方法签名，增加6对时间范围参数：

```java
Page<ProductionPlan> queryProductionPlans(
    ProductionPlan productionPlan,
    LocalDateTime createdTimeStart, LocalDateTime createdTimeEnd,
    LocalDateTime updatedTimeStart, LocalDateTime updatedTimeEnd,
    LocalDateTime productionStartTimeStart, LocalDateTime productionStartTimeEnd,
    LocalDateTime productionEndTimeStart, LocalDateTime productionEndTimeEnd,
    LocalDateTime inspectionStartTimeStart, LocalDateTime inspectionStartTimeEnd,
    LocalDateTime inspectionEndTimeStart, LocalDateTime inspectionEndTimeEnd,
    LocalDateTime outboundTimeStart, LocalDateTime outboundTimeEnd,
    LocalDateTime archiveTimeStart, LocalDateTime archiveTimeEnd,
    int pageNum, int pageSize);
```

同样修改 `searchWithDetails` 方法签名。

### 4. Service 实现: `ProductionPlanServiceImpl.java`
**文件**: `src/main/java/com/tonghui/erp/Service/impl/ProductionPlanServiceImpl.java`

在 `queryProductionPlans` 的 QueryWrapper 构建中，为6个新时间字段各添加 `.ge` / `.le` 条件（null 检查），模式与现有 `created_time` / `update_time` 一致：

```java
if (productionStartTimeStart != null) {
    wrapper.ge("production_start_time", productionStartTimeStart);
}
if (productionStartTimeEnd != null) {
    wrapper.le("production_start_time", productionStartTimeEnd);
}
// ... 其余5个字段同理
```

### 5. Controller: `ProductionPlanController.java`
**文件**: `src/main/java/com/tonghui/erp/Controller/ProductionPlanController.java`

修改以下3个接口，添加12个时间参数（6对 Start/End）：

- `GET /api/production-plans` (`getAllData` 无需修改，传 null)
- `GET /api/production-plans/search`
- `GET /api/production-plans/search-with-details`

### 6. DDL: 新增字段到数据库
**文件**: `db/schema_latest.sql`

在 `production_plan` 表定义中添加6个字段。同时提供 ALTER TABLE SQL：

```sql
ALTER TABLE production_plan
    ADD COLUMN `production_start_time` DATETIME DEFAULT NULL COMMENT '生产开始时间' AFTER `total_amount`,
    ADD COLUMN `production_end_time` DATETIME DEFAULT NULL COMMENT '生产结束时间' AFTER `production_start_time`,
    ADD COLUMN `inspection_start_time` DATETIME DEFAULT NULL COMMENT '检验开始时间' AFTER `production_end_time`,
    ADD COLUMN `inspection_end_time` DATETIME DEFAULT NULL COMMENT '检验结束时间' AFTER `inspection_start_time`,
    ADD COLUMN `outbound_time` DATETIME DEFAULT NULL COMMENT '出库时间' AFTER `inspection_end_time`,
    ADD COLUMN `archive_time` DATETIME DEFAULT NULL COMMENT '归档时间' AFTER `outbound_time`;
```
