# Plan: 文件管理器后端设计

## 需求
前端实现类似 Windows 文件管理器的界面，后端需要支持完整的文件操作。

## 核心功能

| 功能 | 说明 |
|------|------|
| 浏览目录 | 列出指定路径下的文件和文件夹 |
| 新建文件夹 | 在指定路径下创建新目录 |
| 重命名 | 重命名文件或文件夹 |
| 移动 | 移动文件/文件夹到目标路径 |
| 复制 | 复制文件/文件夹到目标路径 |
| 删除 | 删除文件或文件夹 |
| 上传 | 上传文件到指定目录 |
| 下载 | 下载文件 |
| 预览 | 预览文件内容（图片、PDF等） |
| 搜索 | 按文件名搜索 |

## API 设计

### 新建 FileManagerController
**文件**: `src/main/java/com/tonghui/erp/Controller/FileManagerController.java`

```
GET  /api/file-manager/list?path=          — 列出目录内容
POST /api/file-manager/mkdir               — 新建文件夹
PUT  /api/file-manager/rename              — 重命名
PUT  /api/file-manager/move                — 移动
POST /api/file-manager/copy                — 复制
DELETE /api/file-manager/delete            — 删除
POST /api/file-manager/upload              — 上传文件到指定目录
GET  /api/file-manager/download?path=      — 下载文件
GET  /api/file-manager/preview?path=       — 预览文件
GET  /api/file-manager/search?keyword=&path= — 搜索文件
```

## 安全考虑
- 路径遍历防护：禁止 `..` 访问上级目录
- 基路径限制：所有操作必须在 `uploaded-files` 目录内
- 文件类型限制：复用 `FileStorageConfig.allowedTypes`
