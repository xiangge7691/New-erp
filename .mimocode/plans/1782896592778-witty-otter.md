# Plan: 为所有有子表的表添加 search-with-details 接口

## 背景
项目中已有 8 个 `search-with-details` 接口（PurchaseOrders、StockIn、StockOut、Preparation、ProductionPlan、ProductionUnit、ApprovalWorkflow、ApprovalInstance）。现需为剩余 6 个有子表的父表添加相同模式的接口，避免前端多次调用。

## 已有 search-with-details（不需要改动）
| 父表 | 子表 | DTO |
|------|------|-----|
| PurchaseOrders | PurchaseOrderItems | PurchaseOrdersWithItemsDto |
| StockIn | StockInDetail | StockInWithDetailsDto |
| StockOut | StockOutDetail | StockOutWithDetailsDto |
| Preparation | PreparationFormula, PreparationDocument, PreparationProcessTemplate | PreparationWithDetailsDto |
| ProductionPlan | ProductionProcessRecord | ProductionPlanWithRecordsDto |
| ProductionUnit | ProdUnitInvoice, ProdUnitMaterialFile | ProductionUnitWithDetailsDto |
| ApprovalWorkflow | ApprovalNode | ApprovalWorkflowWithNodesDto |
| ApprovalInstance | ApprovalRecord | ApprovalInstanceWithRecordsDto |

## 实现模式（严格遵循现有代码）

### DTO 模式
```java
@Data
@EqualsAndHashCode(callSuper = true)
public class XxxWithDetailsDto extends ParentEntity {
    private List<ChildEntity> children;
}
```

### Service 接口
```java
Page<Parent> queryXxxs(Parent parent, int pageNum, int pageSize);
PagedResult<XxxWithDetailsDto> searchWithDetails(Parent parent, int pageNum, int pageSize);
```

### Service 实现
1. 调用 `queryXxxs()` 获取分页父表数据
2. 收集所有父表 ID
3. 用 `QueryWrapper.in("parent_id", parentIds)` 批量查询每个子表
4. 按父表 ID 分组为 Map
5. `BeanUtils.copyProperties` + 设置子表列表 → 构建 DTO
6. 返回 `PagedResult`

### Controller
```java
@GetMapping("/search-with-details")
public ApiResponse<PagedResult<XxxWithDetailsDto>> searchWithDetails(
        Xxx entity, @ModelAttribute PageRequestDto pageRequestDto) {
    try {
        PagedResult<XxxWithDetailsDto> result = xxxService.searchWithDetails(
            entity, pageRequestDto.getPageIndex(), pageRequestDto.getPageSize());
        return success(result);
    } catch (Exception ex) {
        return exception(ex, "searchWithDetails");
    }
}
```
