4份设计文档：首页设计、人员档案、车间信息、设备管理
对应3个HTML原型：首页.html、人员档案.html、车间详情.html

## 二、需要修改的内容（按优先级排序）

### 🔴 高优先级 — 车间信息模块

#### 2.1 RoomInfo Entity 补充5个字段
- 文件: `src/main/java/com/tonghui/erp/Data/Entity/RoomInfo.java`
- 新增字段:
  - `production_type` (String) — 生产制剂类型，支持多选（内服/外用/眼用）
  - `dust_room` (Boolean) — 产尘操作间
  - `clean_area` (Boolean) — 洁净区
  - `clean_grade` (String) — 洁净等级（A级/B级/C级/D级，洁净区=是时填写）
  - `clean_procedure` (Long) — 洁净规程文件ID，关联file_info

#### 2.2 新建消毒管理记录子表
- 新增 Entity: `DisinfectionRecord.java`
  - 字段: id, room_id, disinfection_date, disinfection_method, disinfection_person, disinfection_cycle(天), next_disinfection_date, remark, created_at, updated_at, is_deleted
- 新增 Mapper + XML
- 新增 Service + Controller
- API: `/api/room/{roomId}/disinfection`

#### 2.3 新建洁净检测记录子表
- 新增 Entity: `CleanInspectionRecord.java`
  - 字段: id, room_id, inspection_date, inspection_area, inspection_item, inspection_result(合格/不合格), inspector, report_file_id, created_at, updated_at, is_deleted
- 新增 Mapper + XML
- 新增 Service + Controller
- API: `/api/room/{roomId}/inspection`

#### 2.4 新建温湿度记录子表（硬件预留）
- 新增 Entity: `TemperatureHumidityRecord.java`
  - 字段: id, room_id, record_date, inspection_area, temperature, humidity, recorder, created_at, is_deleted
- 新增 Mapper + XML
- 新增 Service + Controller
- API: `/api/room/{roomId}/temperature-humidity`

#### 2.5 新建压差记录子表（硬件预留）
- 新增 Entity: `PressureDifferenceRecord.java`
  - 字段: id, room_id, record_date, inspection_area, pressure_value, recorder, created_at, is_deleted
- 新增 Mapper + XML
- 新增 Service + Controller
- API: `/api/room/{roomId}/pressure`

#### 2.6 新增消毒到期提醒接口
- 在 `RoomInfoController` 或新建 `DisinfectionController` 中添加
- API: `GET /api/room/disinfection/reminder?days=30`
- 查询逻辑: 所有启用车间中，下次消毒时间在今天到指定天数之间的记录

### 🟡 中优先级 — 首页模块

#### 2.7 DashboardController 待办事项 — 补充"待入库"和"待确认"
- 文件: `src/main/java/com/tonghui/erp/Controller/DashboardController.java`
- 在 `getTodos()` 方法中新增:
  - **待入库**: 查询 StockIn 表中 `inStatus` 为待入库状态的记录
  - **待确认**: 查询 StockIn 表中 `inStatus` 为待确认状态的记录

#### 2.8 DashboardController — 环境管理待办（占位→实现）
- 当前硬编码 `typeCounts.put("环境管理", 0L)`
- 需要:
  - 在 RoomInfo 表新增资质到期字段（如 `qualification_expire_date`），或新建独立的环境资质表
  - 实现查询逻辑并替换硬编码

#### 2.9 DashboardController — 库存资金占用环形图修复
- 当前逻辑: 按 `quantity` 汇总（数量）
- 设计要求: 按 `quantity × unit_price` 汇总（资金占用金额，单位万元）
- 修改 `getChartData()` 方法中的 `inventoryFundOccupation` 计算逻辑

#### 2.10 DashboardController — 订单跟踪字段补全
- `OrderTrackingDto.batchNo`: 在 `getOrderTracking()` 中从 ProductionPlan 提取批号赋值
- `OrderTrackingDto.hospital`: 在 `getOrderTracking()` 中从 ProductionPlan 提取所属医疗机构赋值

### 🟢 低优先级 — XML修复

#### 2.11 RoomInfoMapper.xml 与 Entity 字段对齐
- 文件: `src/main/resources/Data/mapper/RoomInfoMapper.xml`
- 修复: XML resultMap 中 `creator_name` → `creator_id`，补充 `updater_id`、`updated_time` 映射
- 补充新增5个字段的映射

#### 2.12 EquipmentMapper.xml 补充字段映射
- 文件: `src/main/resources/Data/mapper/EquipmentMapper.xml`
- 补充: `equipment_type`、`maintenance_cycle`、`reminder_days`、`next_maintenance_date` 字段映射

#### 2.13 新建 EquipmentMaintenanceMapper.xml
- 新增文件: `src/main/resources/Data/mapper/EquipmentMaintenanceMapper.xml`
- 包含 BaseResultMap 和 Base_Column_List

## 五、修改顺序建议

1. **RoomInfo Entity + XML** — 补充5个字段（影响车间和设备两个模块）
2. **4个子表 Entity + Mapper + XML + Service + Controller** — 车间子表完整实现
3. **消毒到期提醒接口** — 首页待办数据源
4. **DashboardController** — 待入库/待确认/环境管理待办 + 库存资金图表修复 + 订单跟踪字段补全
5. **XML修复** — RoomInfoMapper.xml、EquipmentMapper.xml、新建EquipmentMaintenanceMapper.xml

## 六、验证方式

1. 启动应用 `./mvnw spring-boot:run`，确认无编译错误
2. 通过 API 测试各新增接口:
   - `POST /api/room` 新增含新字段的车间
   - `POST /api/room/1/disinfection` 新增消毒记录
   - `GET /api/room/disinfection/reminder?days=30` 验证消毒提醒
   - `GET /api/dashboard/todos` 验证待办事项包含所有6类
   - `GET /api/dashboard/charts` 验证库存资金占用图返回金额
   - `GET /api/dashboard/order-tracking` 验证 batchNo 和 hospital 字段有值
3. 运行测试: `./mvnw test`
