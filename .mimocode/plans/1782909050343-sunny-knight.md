# RoomInfo 移除冗余文件字段，改用文件绑定系统

## 问题分析

RoomInfo 实体有 3 个文件相关字段：
- `cleanProcedureFileId` (Long) — 洁净规程文件ID
- `disinfectionProcedure` (String) — 消毒规程文件名
- `cleanInspectionProcedure` (String) — 洁净检测规程文件名

但文件绑定系统已通过 `businessType` + `businessId` 实现了文件关联，FileController 已定义了房间相关的业务类型：
- `ROOM_CLEAN_INSPECTION` → 车间环境/洁净检测
- `ROOM_DISINFECTION` → 车间环境/消毒记录
- `ROOM_TEMPERATURE_HUMIDITY` → 车间环境/温湿度记录
- `ROOM_PRESSURE_DIFFERENCE` → 车间环境/压差记录

**结论**：这 3 个字段是冗余的，应移除，统一使用文件绑定系统管理文件。

## 修改方案

### 1. `src/main/java/com/tonghui/erp/Data/Entity/RoomInfo.java`
移除以下 3 个字段（第 67-71 行、第 93-95 行、第 105-107 行）：
- `cleanProcedureFileId`
- `disinfectionProcedure`
- `cleanInspectionProcedure`

### 2. `db/fix_room_info_drop_file_columns.sql`（新建）
```sql
ALTER TABLE `room_info`
  DROP COLUMN `clean_procedure_file_id`,
  DROP COLUMN `disinfection_procedure`,
  DROP COLUMN `clean_inspection_procedure`;
```

### 3. 更新 schema 文件（可选，保持一致）
- `db/schema_final.sql` — 移除 `clean_procedure_file_id` 列
- `db/schema_latest.sql` — 同上

## 验证

1. `./mvnw compile` 确认编译通过
2. 在生产数据库执行迁移 SQL
3. 重启应用，验证 RoomInfo 的 CRUD 接口正常工作
4. 文件上传仍可通过 `/api/files/upload-with-business` 接口，传 `businessType=ROOM_DISINFECTION&businessId=房间ID` 关联文件
