# 生产计划列表 /api/production-plans/search 添加时间筛选功能

## 现状分析

前端搜索栏有 4 个筛选条件：
| 前端字段 | 后端字段 | 当前支持 |
|---|---|---|
| 计划编号 | `planNumber` | ✅ 已有 LIKE 模糊查询 |
| 制剂名称 | `preparationName` | ✅ 已有 LIKE 模糊查询 |
| 当前状态 | `currentStatus` | ✅ 已有 EQ 精确查询 |
| 时间筛选 | `timeFieldType` + `timeStart` + `timeEnd` | ❌ 缺失 |

**计划编号、制剂名称、当前状态已实现，无需修改。**

## 新增功能：动态时间筛选

前端"时间筛选"是一个下拉框 + 日期范围选择器，用户选择时间类型后，按该类型的时间字段做范围查询。

### 修改文件

#### 1. `src/main/java/com/tonghui/erp/Controller/ProductionPlanController.java`
- 在 `queryProductionPlans` 方法参数中添加 3 个新参数：
  - `@RequestParam(required = false) String timeFieldType` — 时间字段类型（如 `CREATED_TIME`、`PRODUCTION_START_TIME` 等）
  - `@RequestParam(required = false) LocalDateTime timeStart` — 时间起始
  - `@RequestParam(required = false) LocalDateTime timeEnd` — 时间结束
- 将这 3 个参数传递给 service 层

#### 2. `src/main/java/com/tonghui/erp/Service/ProductionPlanService.java`
- 接口方法 `queryProductionPlans` 签名增加 `String timeFieldType, LocalDateTime timeStart, LocalDateTime timeEnd` 参数

#### 3. `src/main/java/com/tonghui/erp/Service/impl/ProductionPlanServiceImpl.java`
- 实现方法签名同步更新
- 在现有时间范围查询逻辑之后，添加动态时间筛选逻辑：
```java
if (StringUtils.hasText(timeFieldType) && (timeStart != null || timeEnd != null)) {
    String column = switch (timeFieldType) {
        case "CREATED_TIME" -> "created_time";
        case "UPDATED_TIME" -> "updated_time";
        case "PRODUCTION_START_TIME" -> "production_start_time";
        case "PRODUCTION_END_TIME" -> "production_end_time";
        case "INSPECTION_START_TIME" -> "inspection_start_time";
        case "INSPECTION_END_TIME" -> "inspection_end_time";
        case "OUTBOUND_TIME" -> "outbound_time";
        case "ARCHIVE_TIME" -> "archive_time";
        default -> null;
    };
    if (column != null) {
        if (timeStart != null) wrapper.ge(column, timeStart);
        if (timeEnd != null) wrapper.le(column, timeEnd);
    }
}
```

#### 4. 同步修改 `searchWith-details` 端点
- `ProductionPlanController.searchWithDetails` 同样添加这 3 个参数
- `ProductionPlanService.searchWithDetails` 接口签名同步
- `ProductionPlanServiceImpl.searchWithDetails` 实现同步

### 前端调用示例

```
GET /api/production-plans/search?pageIndex=0&pageSize=20&planNumber=P2024&currentStatus=1&timeFieldType=CREATED_TIME&timeStart=2024-01-01T00:00:00&timeEnd=2024-12-31T23:59:59
```
