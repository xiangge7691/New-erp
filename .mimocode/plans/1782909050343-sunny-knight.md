# 生产计划状态改为根据时间节点自动计算

## 需求

状态不再手动设置，而是根据时间字段自动推导：
| 状态 | 判断条件 |
|---|---|
| 已下单 (PLAN_ISSUED) | `created_time` 不为空 |
| 生产中 (IN_PRODUCTION) | `production_start_time` 不为空 |
| 已生产 (PRODUCED) | `production_end_time` 不为空 |
| 检验中 (IN_INSPECTION) | `inspection_start_time` 不为空 |
| 已检验 (INSPECTED) | `inspection_end_time` 不为空 |
| 已出库 (OUTBOUND) | `outbound_time` 不为空 |
| 已归档 (ARCHIVED) | `archive_time` 不为空 |

**取最后一个非空时间字段对应的状态**。

## 实施方案

### 1. `ProductionPlanServiceImpl.java` — 新增 `computeStatus` 方法

```java
private String computeStatus(ProductionPlan plan) {
    if (plan.getArchiveTime() != null) return "ARCHIVED";
    if (plan.getOutboundTime() != null) return "OUTBOUND";
    if (plan.getInspectionEndTime() != null) return "INSPECTED";
    if (plan.getInspectionStartTime() != null) return "IN_INSPECTION";
    if (plan.getProductionEndTime() != null) return "PRODUCED";
    if (plan.getProductionStartTime() != null) return "IN_PRODUCTION";
    return "PLAN_ISSUED";
}
```

### 2. `ProductionPlanServiceImpl.java` — 修改 `queryProductionPlans`

将 `wrapper.eq("current_status", ...)` 改为用 SQL CASE 表达式计算状态：

```java
// 添加计算状态列
wrapper.select("*, CASE " +
    "WHEN archive_time IS NOT NULL THEN 'ARCHIVED' " +
    "WHEN outbound_time IS NOT NULL THEN 'OUTBOUND' " +
    "WHEN inspection_end_time IS NOT NULL THEN 'INSPECTED' " +
    "WHEN inspection_start_time IS NOT NULL THEN 'IN_INSPECTION' " +
    "WHEN production_end_time IS NOT NULL THEN 'PRODUCED' " +
    "WHEN production_start_time IS NOT NULL THEN 'IN_PRODUCTION' " +
    "ELSE 'PLAN_ISSUED' END AS computed_status");

// currentStatus 筛选改为匹配计算状态
if (StringUtils.hasText(productionPlan.getCurrentStatus())) {
    wrapper.apply("CASE " +
        "WHEN archive_time IS NOT NULL THEN 'ARCHIVED' " +
        "WHEN outbound_time IS NOT NULL THEN 'OUTBOUND' " +
        "WHEN inspection_end_time IS NOT NULL THEN 'INSPECTED' " +
        "WHEN inspection_start_time IS NOT NULL THEN 'IN_INSPECTION' " +
        "WHEN production_end_time IS NOT NULL THEN 'PRODUCED' " +
        "WHEN production_start_time IS NOT NULL THEN 'IN_PRODUCTION' " +
        "ELSE 'PLAN_ISSUED' END = {0}", productionPlan.getCurrentStatus());
}
```

### 3. `ProductionPlanServiceImpl.java` — 修改 `changePlanStatus`

不再手动设置 `currentStatus`，改为设置对应的时间字段：
- `IN_PRODUCTION` → 设置 `productionStartTime`
- `PRODUCED` → 设置 `productionEndTime`
- `IN_INSPECTION` → 设置 `inspectionStartTime`
- `INSPECTED` → 设置 `inspectionEndTime`
- `OUTBOUND` → 设置 `outboundTime`
- `ARCHIVED` → 设置 `archiveTime`

移除 `validateStatusChange` 调用，改为校验时间字段的先后顺序。

### 4. `DashboardController.java` — 修改统计查询

将 `wrapper.eq("current_status", ...)` 改为用 SQL CASE 表达式：

```java
// 计算状态的 SQL 片段
String statusExpr = "CASE " +
    "WHEN archive_time IS NOT NULL THEN 'ARCHIVED' " +
    "WHEN outbound_time IS NOT NULL THEN 'OUTBOUND' " +
    "WHEN inspection_end_time IS NOT NULL THEN 'INSPECTED' " +
    "WHEN inspection_start_time IS NOT NULL THEN 'IN_INSPECTION' " +
    "WHEN production_end_time IS NOT NULL THEN 'PRODUCED' " +
    "WHEN production_start_time IS NOT NULL THEN 'IN_PRODUCTION' " +
    "ELSE 'PLAN_ISSUED' END";

// 统计查询
productionStats.setInProgress(productionPlanService.count(
    new QueryWrapper<ProductionPlan>().apply(statusExpr + " IN ({0})",
        "IN_PRODUCTION", "PRODUCED", "IN_INSPECTION", "INSPECTED")));
```
