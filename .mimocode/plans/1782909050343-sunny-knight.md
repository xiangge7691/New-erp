# 修复 production_process_record 表缺少 is_deleted 列的问题

## 问题分析

错误信息：`Unknown column 'is_deleted' in 'field list'`

**根因**：`ProductionProcessRecord.java` 实体类声明了 `isDeleted` 字段（第125-126行），映射到 `is_deleted` 列，但生产数据库的 `production_process_record` 表实际上**没有这个列**。

对比：
- **Entity** (`src/main/java/com/tonghui/erp/Data/Entity/ProductionProcessRecord.java:125-126`)：有 `isDeleted` 字段
- **数据库 dump** (`db/_192_168_5_100-2026_07_01_14_38_19-dump.sql:885-921`)：表中没有 `is_deleted` 和 `version` 列
- **schema_final.sql / schema_latest.sql**：定义了这两个列，但未同步到生产库

同样缺少的还有 `version` 列（乐观锁），Entity 中也有（第131-132行）。

## 修复方案

### 方案：ALTER TABLE 添加缺失列（推荐）

在数据库中执行以下 SQL，添加缺失的 `is_deleted` 和 `version` 列：

```sql
ALTER TABLE `production_process_record` 
  ADD COLUMN `is_deleted` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否已删除' AFTER `remark`,
  ADD COLUMN `version` INT NOT NULL DEFAULT 0 COMMENT '乐观锁版本号' AFTER `is_deleted`;
```

### 需要修改的文件

1. **新建迁移 SQL 文件** `db/fix_production_process_record_missing_columns.sql`
   - 添加 `ALTER TABLE` 语句为 `production_process_record` 表补充 `is_deleted` 和 `version` 列

2. **无需修改 Java 代码** — Entity 和 Mapper 代码正确，问题仅在于数据库缺少列。

## 验证

1. 在生产数据库上执行迁移 SQL
2. 重启应用，调用获取生产计划工序记录的接口，确认不再报错
3. 验证 `is_deleted` 默认值为 0（未删除），`version` 默认值为 0
