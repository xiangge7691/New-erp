# 为所有有子表的实体添加 search 接口（返回子表数据）

## 目标

为所有有子表的父表实体添加 search 接口，返回结果中包含子表数据。

## 涉及的父子关系（共8组）

| # | 父表 | 子表 | 外键字段 | 父Controller |
|---|------|------|----------|---------------|
| 1 | PurchaseOrders | PurchaseOrderItems | order_id | PurchaseOrdersController `/api/purchase-orders` |
| 2 | StockIn | StockInDetail | in_id | StockInController `/api/stockin` |
| 3 | StockOut | StockOutDetail | out_id | StockOutController `/api/stockout` |
| 4 | ProductionUnit | ProdUnitInvoice + ProdUnitMaterialFile | prod_unit_id | ProductionUnitController `/api/production_unit` |
| 5 | ProductionPlan | ProductionProcessRecord | plan_id | ProductionPlanController `/api/production-plans` |
| 6 | Preparation | PreparationFormula + PreparationDocument + PreparationProcessTemplate | preparation_id | PreparationController `/api/preparation` |
| 7 | ApprovalWorkflow | ApprovalNode | workflow_id | ApprovalWorkflowController `/api/approval/workflow` |
| 8 | ApprovalInstance | ApprovalRecord | instance_id | ApprovalInstanceController `/api/approval/instance` |

> 注意：Role → RolePerm 和 User → UserRole/UserDepartment 已有 RoleDto/UserDto 包含子表数据，不需要新增。

## 实现方案

对每组父子关系，按以下步骤新增：

### 1. 新建 DTO 类
在 `Common/Dto` 对应子目录下新建包含子表的 DTO，继承父实体并添加子表列表字段。

### 2. Service 接口新增方法
```java
PagedResult<XxxWithDetailsDto> searchWithDetails(Xxx query, int pageIndex, int pageSize);
```

### 3. Service 实现类实现该方法
- 先调用已有的 queryXxx 方法分页查询父表
- 再批量查询子表（通过父ID列表 IN 查询）
- 组装成 DTO 返回

### 4. Controller 新增 search-with-details 端点（保留现有 /search 不变）
```java
@GetMapping("/search-with-details")
public ApiResponse<PagedResult<XxxWithDetailsDto>> searchWithDetails(...)
```

## 通用实现模式

### DTO 模式
```java
@Data
@EqualsAndHashCode(callSuper = true)
public class XxxWithDetailsDto extends Xxx {
    private List<ChildEntity> details;
}
```

### Service 实现模式
```java
public PagedResult<XxxWithDetailsDto> searchWithDetails(Xxx query, int pageIndex, int pageSize) {
    // 1. 分页查询父表
    Page<Xxx> parentPage = queryXxxs(query, pageIndex, pageSize);
    List<Xxx> parents = parentPage.getRecords();
    
    if (parents.isEmpty()) {
        return new PagedResult<>();
    }
    
    // 2. 提取父表ID列表
    List<Long> parentIds = parents.stream().map(Xxx::getId).collect(Collectors.toList());
    
    // 3. 批量查询子表
    QueryWrapper<Child> wrapper = new QueryWrapper<>();
    wrapper.in("parent_id_field", parentIds);
    List<Child> allChildren = childService.list(wrapper);
    
    // 4. 按父ID分组
    Map<Long, List<Child>> childrenMap = allChildren.stream()
        .collect(Collectors.groupingBy(Child::getParentIdField));
    
    // 5. 组装DTO
    List<XxxWithDetailsDto> dtos = parents.stream().map(parent -> {
        XxxWithDetailsDto dto = new XxxWithDetailsDto();
        BeanUtils.copyProperties(parent, dto);
        dto.setDetails(childrenMap.getOrDefault(parent.getId(), List.of()));
        return dto;
    }).collect(Collectors.toList());
    
    PagedResult<XxxWithDetailsDto> result = new PagedResult<>();
    result.setItems(dtos);
    result.setTotalCount(parentPage.getTotal());
    result.setPageIndex(pageIndex);
    result.setPageSize(pageSize);
    return result;
}
```

### Controller 端点模式
```java
@GetMapping("/search-with-details")
public ApiResponse<PagedResult<XxxWithDetailsDto>> searchWithDetails(
        Xxx query,
        @RequestParam int pageIndex,
        @RequestParam int pageSize) {
    try {
        int safePageIndex = Math.max(0, pageIndex);
        int safePageSize = pageSize <= 0 ? 20 : Math.max(1, pageSize);
        PagedResult<XxxWithDetailsDto> result = xxxService.searchWithDetails(query, safePageIndex, safePageSize);
        return success(result);
    } catch (Exception ex) {
        return exception(ex, "查询失败");
    }
}
```
