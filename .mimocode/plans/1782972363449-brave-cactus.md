## 三、Common模块调整

### 保留在 Common/ 根目录
- `Config/`: 全部保留（MybatisPlusConfig, MybatisPlusMetaObjectHandler, SecurityConfig, JwtConfig, JwtAuthenticationFilter, FileStorageConfig）
- `Dto/`: ApiResponse.java, PageRequestDto.java, PagedResult.java
- `exception/`: GlobalExceptionHandler.java
- `Mapper/`: Converters.java → 移至 Data/mapper/system/
- `utils/`: 全部保留（PasswordHasher, JwtHelper, EntityUtils, JsonFormatter, DataExporterImporter, Base64）

### 新增异常类
在 `Common/exception/` 下添加：
- `BusinessException.java` — 通用业务异常
- `NotFoundException.java` — 资源不存在异常
- `UnauthorizedException.java` — 未授权异常

## 四、注释规范补充

### 1. 类级别注释
- 所有Controller类：模块说明 + API功能描述
- 所有Service接口：业务功能描述
- 所有Entity类：表名 + 业务含义

### 2. 方法级别注释
- Service接口方法：完整Javadoc（@param、@return、@throws）
- Controller方法：API功能说明
- 复杂业务逻辑：行内注释说明"为什么"

### 3. 分组标记
- 使用 `#region` / `#endregion` 分组（保持现有风格）
- 每个分组添加分隔线注释

## 六、模块间依赖关系

```
Dashboard ──fans out to──> Production, Stock, Approval,
                           Equipment, Personnel, Environment

Personnel ──depends on──> System [User, Dept, Position],
                           Environment [RoomInfo],
                           File [FileInfoService]

Equipment ──depends on──> Environment [RoomInfo],
                           File [FileInfoService]

Production ──depends on──> Preparation [Preparation],
                            Master Data [ProductionUnit]

Stock ──depends on──> Master Data [ProductionUnit, Material],
                       Purchase [PurchaseSuppliers] (via StockIn)

Preparation ──depends on──> Master Data [ProcessType, Unit, DosageForm]

Purchase ──binds to──> Approval [ApprovalInstance via relatedType]

Approval ──generic──> any domain (via relatedType + relatedId)
```
