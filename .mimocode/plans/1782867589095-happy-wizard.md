# ERP数据库字段不一致修复计划

## 问题概述

通过对比Entity类与`schema_latest.sql`，发现多个Entity类仍使用旧版字符串字段，而schema已升级为外键关联。核心问题是：

1. **字符串字段 vs 外键ID**：Entity使用`category_name`/`unit_name`等字符串，schema使用`category_id`/`unit_id`外键
2. **时间字段命名不一致**：Entity使用`createdTime`/`updatedTime`，schema使用`created_at`/`updated_at`
3. **缺少软删除/乐观锁字段**：部分Entity缺少`is_deleted`/`version`字段
4. **缺少审批实例ID**：`StockIn`/`StockOut`缺少`approval_instance_id`字段

## 需要修复的Entity类

| Entity类 | 主要问题 |
|----------|----------|
| `ProductionPlan` | `preparationCode`→`preparation_id`, `unitName`→`unit_id`, `productionUnit`→`prod_unit_id`, `createTime`/`updateTime`→`created_at`/`updated_at` |
| `Material` | `categoryName`→`category_id`, `unitName`→`unit_id`, 缺少`is_deleted`/`version` |
| `Preparation` | `unitName`→`unit_id`, `dosageForm`→`dosage_form_id`, `producer`→`prod_unit_id`, 缺少`is_deleted`/`version` |
| `ProductionProcessRecord` | `processName`→`process_type_id`, `operatorName`→`operator_id`, `workshop`→`room_id`, `equipment`→`equipment_id`, `unitName`→`unit_id` |
| `WorkOrder` | `preparationCode`/`preparationName`→`preparation_id`, `producer`→`producer_unit_id`, `receiver`→`receiver_unit_id` |
| `Stock` | `itemCode`/`itemName`→删除, `categoryName`→`category_id`, `unitName`→`unit_id` |
| `StockIn` | `supplierId`类型Long→INT, 缺少`approval_instance_id`, `is_deleted`, `version` |
| `StockInDetail` | `itemCode`/`itemName`→删除, `categoryName`→`category_id`, `unitName`→`unit_id`, 缺少`is_deleted` |
| `StockOut` | 缺少`approval_instance_id`, `is_deleted`, `version` |
| `PreparationDocument` | `fileId`→删除(改用business_file关联) |

## 交付物

1. **最终版数据库脚本** (`db/schema_final.sql`)：基于`schema_latest.sql`，确保与Entity类一致
2. **数据库修复文档** (`docs/数据库字段修复文档.md`)：记录所有变更和修复步骤

## 执行步骤

### 步骤1：创建最终版数据库脚本
- 以`schema_latest.sql`为基础
- 确保所有字段与Entity类定义一致
- 添加缺失的字段和约束

### 步骤2：创建数据库修复文档
- 记录所有Entity与schema的不一致
- 提供ALTER TABLE修复语句
- 记录数据迁移注意事项

## 验证方式

1. 运行最终版SQL脚本，确保无语法错误
2. 对比所有Entity类的`@TableField`注解与表字段
3. 确保所有外键约束正确
