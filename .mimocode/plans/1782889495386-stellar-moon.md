# 人员档案证书子表设计方案

## 需求概述
在 `personnel_file` 表下新增证书子表 `personnel_certificate`，替代原有 `certificate_name`/`certificate_no` 字段，支持一人多证书。

---

## 1. 数据库设计

### 新表 SQL (`db/V3_3__personnel_certificate.sql`)

```sql
CREATE TABLE IF NOT EXISTS `personnel_certificate` (
    `certificate_id`     BIGINT       NOT NULL AUTO_INCREMENT COMMENT '证书唯一标识',
    `personnel_file_id`  BIGINT       NOT NULL COMMENT '关联人员档案ID',
    `certificate_type`   VARCHAR(50)  DEFAULT NULL COMMENT '证书类型（如：执业资格/职称证书/健康证/培训证书/其他）',
    `certificate_name`   VARCHAR(200) NOT NULL COMMENT '证书名称',
    `certificate_no`     VARCHAR(100) DEFAULT NULL COMMENT '证书编号',
    `issuing_authority`  VARCHAR(200) DEFAULT NULL COMMENT '发证机构',
    `issue_date`         DATE         DEFAULT NULL COMMENT '发证日期',
    `expiry_date`        DATE         DEFAULT NULL COMMENT '到期日期',
    `certificate_level`  VARCHAR(50)  DEFAULT NULL COMMENT '证书等级/级别',
    `applicable_scope`   VARCHAR(500) DEFAULT NULL COMMENT '适用范围',
    `original_status`    VARCHAR(20)  DEFAULT NULL COMMENT '原件/复印件',
    `reexamine_date`     DATE         DEFAULT NULL COMMENT '复审日期',
    `training_record`    VARCHAR(500) DEFAULT NULL COMMENT '培训记录',
    `file_id`            BIGINT       DEFAULT NULL COMMENT '附件（关联file_info.file_id）',
    `status`             INT          DEFAULT 1 COMMENT '状态：0失效/1有效',
    `remark`             VARCHAR(500) DEFAULT NULL COMMENT '备注',
    `created_by`         BIGINT       DEFAULT NULL COMMENT '创建人ID',
    `updated_by`         BIGINT       DEFAULT NULL COMMENT '更新人ID',
    `created_time`       DATETIME     DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_time`       DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `is_deleted`         INT          DEFAULT 0 COMMENT '是否已删除：0否/1是',
    `version`            INT          DEFAULT 0 COMMENT '乐观锁版本号',
    PRIMARY KEY (`certificate_id`),
    KEY `idx_personnel_file_id` (`personnel_file_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='人员证书子表';
```

### 原表字段处理
从 `personnel_file` 中移除 `certificate_name` 和 `certificate_no` 字段：
```sql
ALTER TABLE `personnel_file`
    DROP COLUMN `certificate_name`,
    DROP COLUMN `certificate_no`;
```

## 2. 后端设计

### 2.1 Entity: `PersonnelCertificate.java`
路径: `src/main/java/com/tonghui/erp/Data/Entity/PersonnelCertificate.java`

继承 `AuditEntity`，包含所有表字段。

### 2.2 Mapper: `PersonnelCertificateMapper.java`
路径: `src/main/java/com/tonghui/erp/Data/mapper/PersonnelCertificateMapper.java`

继承 `BaseMapper<PersonnelCertificate>`。

### 2.3 Service: `PersonnelCertificateService.java` + `PersonnelCertificateServiceImpl.java`
路径:
- `src/main/java/com/tonghui/erp/Service/PersonnelCertificateService.java`
- `src/main/java/com/tonghui/erp/Service/impl/PersonnelCertificateServiceImpl.java`

核心方法：
- `List<PersonnelCertificate> getByPersonnelFileId(Long personnelFileId)` — 查询某人员所有证书
- `void saveCertificates(Long personnelFileId, List<PersonnelCertificate> certificates)` — 先删后插保存证书列表

### 2.4 Controller: `PersonnelCertificateController.java`
路径: `src/main/java/com/tonghui/erp/Controller/PersonnelCertificateController.java`

REST API:
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/personnelCertificate/personnel/{personnelFileId}` | 查询某人员的所有证书 |
| POST | `/api/personnelCertificate/personnel/{personnelFileId}` | 批量保存某人员的证书（先删后插） |
| POST | `/api/personnelCertificate` | 单独新增一条证书 |
| PUT | `/api/personnelCertificate/{id}` | 修改证书 |
| DELETE | `/api/personnelCertificate/{id}` | 删除证书 |

### 2.5 PersonnelFileController 修改
在人员档案详情查询 (`GET /api/personnelFile/{id}`) 中，返回结果增加 `List<PersonnelCertificate> certificates` 字段，方便前端一次性获取人员及其所有证书。

方案：在 `PersonnelFile` 实体增加 `@TableField(exist = false) List<PersonnelCertificate> certificates`，在 `PersonnelFileController.getById()` 中填充。
