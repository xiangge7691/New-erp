# 房间搜索带子表 (search-with-details) 方案

## 需求概述
为 `RoomInfo` (房间) 添加 `search-with-details` 接口，搜索房间时同时返回关联的 4 个子表记录。

## 房间关联的子表（均有 `room_id` 字段和 `findByRoomId` 方法）
1. `TemperatureHumidityRecord` — 温湿度记录
2. `PressureDifferenceRecord` — 压差记录
3. `CleanInspectionRecord` — 洁净检测记录
4. `DisinfectionRecord` — 消毒记录

---

## 1. DTO 设计

### `RoomInfoWithDetailsDto.java`
路径: `src/main/java/com/tonghui/erp/Common/Dto/Room/RoomInfoWithDetailsDto.java`

继承 `RoomInfo`，包含 4 个子表 List 字段：
```java
@Data
@EqualsAndHashCode(callSuper = true)
public class RoomInfoWithDetailsDto extends RoomInfo {
    private List<TemperatureHumidityRecord> temperatureHumidityRecords;
    private List<PressureDifferenceRecord> pressureDifferenceRecords;
    private List<CleanInspectionRecord> cleanInspectionRecords;
    private List<DisinfectionRecord> disinfectionRecords;
}
```

---

## 2. Service 层修改

### `RoomInfoService.java` — 新增方法
```java
PagedResult<RoomInfoWithDetailsDto> searchWithDetails(String roomName, PageRequestDto pageRequest);
```

### `RoomInfoServiceImpl.java` — 实现
- 复用 `searchByName` 查询房间主表分页
- 批量查询 4 个子表（用 `IN` 查询所有 roomId）
- 按 `roomId` 分组后组装 DTO

需注入 4 个子表 Service：
- `TemperatureHumidityRecordService`
- `PressureDifferenceRecordService`
- `CleanInspectionRecordService`
- `DisinfectionRecordService`

---

## 3. Controller 层修改

### `RoomInfoController.java` — 新增端点
```java
@GetMapping("/search-with-details")
public ApiResponse<PagedResult<RoomInfoWithDetailsDto>> searchWithDetails(
        @RequestParam(required = false) String roomName,
        @ModelAttribute PageRequestDto pageRequest)
```
