# 物料管理模块

## 模块概述

物料管理模块负责管理企业生产所需的各种物料信息，包括原料、辅料、包材等。该模块提供物料的增删改查、分类管理、规格管理等核心功能，是生产管理和采购管理的基础数据支撑。

## 功能特性

- 物料信息管理：物料的创建、编辑、删除、查询
- 物料分类管理：支持原料、辅料、包材等多种分类
- 物料编码管理：唯一编码标识，支持自定义编码规则
- 计量单位管理：支持多种计量单位（kg、张、瓶等）
- 规格描述管理：记录物料的详细规格信息
- 状态管理：支持物料的启用/禁用状态
- 高级查询：支持多条件组合查询和时间段筛选

## 核心接口

### 物料管理接口

**控制器**: [MaterialController.java](../src/main/java/com/tonghui/erp/Controller/MaterialController.java)

#### 1.1 获取物料列表
- **路径**: `GET /api/material`
- **参数**: 
  - `pageIndex`: 页码（从0开始）
  - `pageSize`: 每页数量
- **返回**: `PagedResult<Material>`

#### 1.2 获取物料详情
- **路径**: `GET /api/material/{id}`
- **参数**: `id` - 物料ID
- **返回**: `Material`

#### 1.3 创建物料
- **路径**: `POST /api/material`
- **参数**: `Material`
  - `materialCode`: 物料编码（唯一）
  - `materialName`: 物料名称
  - `categoryName`: 分类名称
  - `unitName`: 计量单位
  - `spec`: 规格描述
  - `materialStatus`: 状态
  - `remark`: 备注
- **返回**: `Material`

#### 1.4 更新物料
- **路径**: `PUT /api/material/{id}`
- **参数**: 
  - `id`: 物料ID
  - `Material`: 物料信息
- **返回**: `Material`

#### 1.5 删除物料
- **路径**: `DELETE /api/material/{id}`
- **参数**: `id` - 物料ID
- **返回**: `boolean`

#### 1.6 高级查询物料
- **路径**: `GET /api/material/search`
- **参数**:
  - `materialName`: 物料名称（模糊匹配）
  - `categoryName`: 分类名称（精确匹配）
  - `unitName`: 计量单位（精确匹配）
  - `spec`: 规格描述（模糊匹配）
  - `materialStatus`: 状态
  - `createdTimeStart`: 创建时间起始
  - `createdTimeEnd`: 创建时间结束
  - `updatedTimeStart`: 更新时间起始
  - `updatedTimeEnd`: 更新时间结束
  - `pageIndex`: 页码
  - `pageSize`: 每页数量
- **返回**: `PagedResult<Material>`

**查询示例**:
```
GET /api/material/search?pageIndex=0&pageSize=20&materialName=瓶&categoryName=包材&unitName=个&materialStatus=1
```

## 业务流程

### 物料创建流程

1. 用户输入物料信息
2. 系统验证物料编码是否唯一
3. 验证必填字段是否完整
4. 保存物料信息到数据库
5. 返回创建的物料信息

### 物料更新流程

1. 用户提交物料更新信息
2. 系统验证物料是否存在
3. 如果更新了物料编码，验证新编码是否唯一
4. 更新物料信息
5. 记录更新人和更新时间
6. 返回更新后的物料信息

### 物料查询流程

1. 用户输入查询条件
2. 系统根据条件构建查询
3. 支持模糊匹配和精确匹配
4. 支持时间段筛选
5. 返回分页查询结果

## 物料分类

系统支持多种物料分类，常见的分类包括：

- **原料**: 生产所需的主要原材料
- **辅料**: 生产过程中使用的辅助材料
- **包材**: 产品包装所需材料
- **耗材**: 生产过程中消耗的材料

## 计量单位

系统支持多种计量单位，常见的单位包括：

- **重量单位**: kg（千克）、g（克）、t（吨）
- **数量单位**: 个、件、套
- **体积单位**: L（升）、ml（毫升）
- **长度单位**: m（米）、cm（厘米）
- **面积单位**: m²（平方米）
- **包装单位**: 箱、盒、包、瓶、张

## 物料状态

- **1**: 启用状态，物料可以使用
- **0**: 禁用状态，物料不可使用

## 业务规则

### 物料编码规则

1. 物料编码必须唯一
2. 建议采用统一的编码规则，如：分类码+流水号
3. 示例：YL-00001（原料类第1个物料）

### 物料命名规范

1. 物料名称应简洁明了
2. 应包含物料的主要特征
3. 避免使用模糊不清的名称

### 物料规格描述

1. 规格描述应详细准确
2. 包含物料的物理特性、化学特性等
3. 示例：500ml，透明，PET材质

## 使用示例

### 创建物料

```json
POST /api/material
{
  "materialCode": "YL-00001",
  "materialName": "葡萄糖",
  "categoryName": "原料",
  "unitName": "kg",
  "spec": "食品级，纯度99%",
  "materialStatus": 1,
  "remark": "主要原料"
}
```

### 查询物料

```http
GET /api/material/search?pageIndex=0&pageSize=20&categoryName=原料&materialStatus=1
```

### 更新物料

```json
PUT /api/material/1
{
  "materialName": "葡萄糖（食品级）",
  "spec": "食品级，纯度99.5%",
  "remark": "主要原料，已更新规格"
}
```

### 删除物料

```http
DELETE /api/material/1
```

## 扩展功能

### 物料导入导出

系统支持物料的批量导入和导出功能，便于数据的快速维护和迁移。

### 物料关联

物料可以与以下模块关联：
- **生产管理**: 制剂配方中的物料
- **采购管理**: 采购订单中的物料
- **库存管理**: 库存中的物料

### 物料审批

可结合审批流程模块，实现物料信息的审批管理。

## 注意事项

1. 物料编码一旦创建，不建议修改，以免影响历史数据
2. 删除物料前，应检查是否有相关的生产订单、采购订单或库存记录
3. 物料状态为禁用时，不应再创建新的采购订单或生产计划
4. 物料规格描述应尽可能详细，以便于采购和使用
