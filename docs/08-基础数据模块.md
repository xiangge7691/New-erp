# 基础数据模块

## 模块概述

基础数据模块负责管理ERP系统中的各类基础数据，包括计量单位、药品剂型等。该模块提供基础数据的增删改查功能，是其他业务模块的基础支撑。

## 功能特性

- 计量单位管理：计量单位的创建、编辑、删除、查询
- 药品剂型管理：药品剂型的创建、编辑、删除、查询
- 模糊查询：支持按名称进行模糊查询
- 分页查询：支持分页查询功能
- 数据验证：确保数据的唯一性和完整性

## 数据表结构

### 1. unit（计量单位表）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| unit_id | BIGINT | 单位唯一标识 | 主键，自增 |
| unit_name | VARCHAR | 单位中文名称（法定全称） | 唯一 |
| symbol | VARCHAR | 单位符号（区分大小写） | - |

**实体类**: [Unit.java](../src/main/java/com/tonghui/erp/Data/Entity/Unit.java)

### 2. dosage_form（药品剂型分类表）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| dosage_id | BIGINT | 剂型唯一标识 | 主键，自增 |
| dosage_name | VARCHAR | 剂型名称（法定全称） | 唯一 |
| remark | VARCHAR | 剂型特性备注 | - |

**实体类**: [DosageForm.java](../src/main/java/com/tonghui/erp/Data/Entity/DosageForm.java)

## 核心接口

### 1. 计量单位管理接口

**控制器**: [UnitController.java](../src/main/java/com/tonghui/erp/Controller/UnitController.java)

#### 1.1 获取计量单位列表
- **路径**: `GET /api/Unit`
- **参数**: 
  - `pageIndex`: 页码（从0开始）
  - `pageSize`: 每页数量
- **返回**: `PagedResult<Unit>`

#### 1.2 获取计量单位详情
- **路径**: `GET /api/Unit/{id}`
- **参数**: `id` - 计量单位ID
- **返回**: `Unit`

#### 1.3 创建计量单位
- **路径**: `POST /api/Unit`
- **参数**: `Unit`
  - `unitName`: 单位中文名称（法定全称）
  - `symbol`: 单位符号（区分大小写）
- **返回**: `Unit`

#### 1.4 更新计量单位
- **路径**: `PUT /api/Unit/{id}`
- **参数**: 
  - `id`: 计量单位ID
  - `Unit`: 计量单位信息
- **返回**: `Unit`

#### 1.5 删除计量单位
- **路径**: `DELETE /api/Unit/{id}`
- **参数**: `id` - 计量单位ID
- **返回**: `boolean`

#### 1.6 搜索计量单位
- **路径**: `GET /api/Unit/search`
- **参数**:
  - `unitName`: 计量单位名称关键词（模糊匹配）
  - `pageIndex`: 页码
  - `pageSize`: 每页数量
- **返回**: `PagedResult<Unit>`

## 业务流程

### 计量单位创建流程

1. 用户输入计量单位信息
2. 系统验证单位名称是否唯一
3. 保存计量单位信息
4. 返回创建的计量单位信息

### 药品剂型创建流程

1. 用户输入药品剂型信息
2. 系统验证剂型名称是否唯一
3. 保存药品剂型信息
4. 返回创建的药品剂型信息

## 业务规则

### 计量单位规则

1. 计量单位名称必须唯一
2. 计量单位符号区分大小写
3. 计量单位名称应为法定全称
4. 删除计量单位前，应检查是否被其他数据引用

### 药品剂型规则

1. 药品剂型名称必须唯一
2. 药品剂型名称应为法定全称
3. 删除药品剂型前，应检查是否被其他数据引用

## 使用示例

### 创建计量单位

```json
POST /api/Unit
{
  "unitName": "千克",
  "symbol": "kg"
}
```

### 创建药品剂型

```json
POST /api/DosageForm
{
  "dosageName": "片剂",
  "remark": "口服片剂"
}
```

### 搜索计量单位

```http
GET /api/Unit/search?unitName=克&pageIndex=0&pageSize=20
```

### 搜索药品剂型

```http
GET /api/DosageForm/search?dosageFormName=片&pageIndex=0&pageSize=20
```

## 注意事项

1. 计量单位和药品剂型一旦被其他数据引用，不建议删除
2. 计量单位名称和符号应符合国家标准
3. 药品剂型名称应为法定全称
4. 基础数据是系统运行的基础，修改时应谨慎
5. 删除基础数据前，应检查是否有相关的业务数据引用
