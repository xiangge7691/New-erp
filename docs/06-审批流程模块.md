# 审批流程模块

## 模块概述

审批流程模块是ERP系统的核心模块之一，负责管理企业各类业务的审批流程。该模块实现了灵活的工作流定义、审批实例管理、审批节点配置和审批记录跟踪等功能，支持多级审批、并行审批、转交驳回等复杂的审批场景。

## 功能特性

- 工作流定义：创建、编辑、删除审批流程
- 审批节点配置：配置审批节点、审批角色、节点顺序
- 审批实例管理：创建、查询、更新审批实例
- 审批记录跟踪：记录所有审批操作，支持审计追溯
- 审批动作支持：同意、驳回、转交
- 业务关联：支持关联采购订单、生产计划等业务单据
- 角色权限：基于角色的审批权限控制
- 状态管理：待审批、已同意、已驳回、已转交、已归档

## 业务流程

### 审批流程创建流程

1. 创建审批流程定义
2. 添加审批节点
3. 配置节点审批角色
4. 设置节点顺序
5. 配置节点提示和状态

### 审批实例创建流程

1. 根据业务类型选择审批流程
2. 创建审批实例
3. 关联业务单据
4. 初始化当前节点为第一个节点
5. 设置实例状态为待审批

### 审批执行流程

1. 用户查看待审批列表
2. 用户选择审批实例
3. 用户执行审批操作（同意/驳回/转交）
4. 系统记录审批记录
5. 系统更新审批实例状态
6. 如果同意，流转到下一节点
7. 如果驳回，返回到上一节点或结束
8. 如果转交，流转到指定节点

## 审批动作

### 同意（AGREE）
- 审批人同意当前节点的审批
- 流程流转到下一节点
- 如果是最后一个节点，流程结束，状态变更为已同意

### 驳回（REJECT）
- 审批人驳回当前节点的审批
- 流程返回到上一节点或结束
- 实例状态变更为已驳回
- 需要填写驳回原因

### 转交（TRANSFER）
- 审批人将审批转交给其他节点
- 流程流转到指定节点
- 需要选择转交目标节点
- 需要填写转交原因

## 审批状态流转

### 正常流转
1. `PENDING`（待审批）→ `PENDING`（下一节点待审批）→ ... → `APPROVED`（已同意）

### 驳回流转
1. `PENDING`（待审批）→ `REJECTED`（已驳回）

### 转交流转
1. `PENDING`（待审批）→ `TRANSFERRED`（已转交）→ `PENDING`（目标节点待审批）

### 归档流转
1. `APPROVED`（已同意）→ `ARCHIVED`（已归档）
2. `REJECTED`（已驳回）→ `ARCHIVED`（已归档）

## 业务规则

### 审批流程规则

1. 一个流程可以有多个节点
2. 节点按顺序执行
3. 每个节点需要配置审批角色
4. 流程类型必须唯一

### 审批实例规则

1. 一个业务单据只能有一个审批实例
2. 审批实例必须关联一个审批流程
3. 审批实例必须关联一个业务单据
4. 审批实例状态不可逆

### 审批记录规则

1. 每次审批操作都生成一条审批记录
2. 审批记录不可删除
3. 审批记录包含审批人、审批动作、审批意见

## 扩展功能

### 并行审批

支持多个节点并行审批，所有节点都通过后才流转到下一节点。

### 条件审批

支持根据业务条件动态选择审批路径。

### 审批超时

支持设置审批超时时间，超时后自动提醒或升级。

### 审批统计

生成审批统计报表，包括审批效率、审批通过率等。

## 注意事项

1. 审批流程定义后，不建议频繁修改
2. 删除审批流程前，应检查是否有正在进行的审批实例
3. 审批记录不可删除，用于审计和追溯
4. 审批实例状态变更需要记录审批记录
5. 节点顺序必须连续，不能有跳跃
6. 审批角色必须存在且有权限
