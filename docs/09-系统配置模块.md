# 系统配置模块

## 模块概述

系统配置模块负责管理ERP系统的各类配置信息，包括JWT配置、安全配置、数据库配置、文件存储配置等。该模块提供系统配置的集中管理，确保系统的安全性和可配置性。

## 功能特性

- JWT配置：JWT令牌的密钥、签发者、受众、有效期等配置
- 安全配置：密码加密、安全过滤等配置
- 数据库配置：数据库连接信息、连接池配置等
- 文件存储配置：文件存储路径、大小限制、类型限制等配置
- MyBatis-Plus配置：分页配置、驼峰命名转换等配置
- 系统初始化：root用户初始化等系统初始化配置

## 配置说明

### JWT配置

#### 配置项
- `jwt.secret-key`: JWT密钥，用于签名和验证JWT令牌
- `jwt.issuer`: JWT签发者标识
- `jwt.audience`: JWT受众标识
- `jwt.expires-in-minutes`: 访问令牌有效期（分钟）
- `jwt.refresh-expires-in-minutes`: 刷新令牌有效期（分钟）

#### 配置建议
1. JWT密钥应使用强随机字符串，长度至少32位
2. 访问令牌有效期建议设置为2小时（120分钟）
3. 刷新令牌有效期建议设置为7天（10080分钟）
4. 签发者和受众应使用有意义的标识符

### 数据库配置

#### 配置项
- `spring.datasource.url`: 数据库连接URL
- `spring.datasource.username`: 数据库用户名
- `spring.datasource.password`: 数据库密码
- `spring.datasource.driver-class-name`: 数据库驱动类名

#### 配置建议
1. 数据库连接URL应包含必要的连接参数
2. 数据库密码应使用强密码
3. 生产环境建议使用连接池
4. 建议启用SSL连接

### MyBatis-Plus配置

#### 配置项
- `mybatis-plus.configuration.map-underscore-to-camel-case`: 是否开启驼峰命名转换
- `mybatis-plus.mapper-locations`: Mapper XML文件位置
- `mybatis-plus.global-config.db-config.pagination`: 是否启用分页

#### 配置建议
1. 建议开启驼峰命名转换，便于Java开发
2. Mapper XML文件位置应与实际文件位置一致
3. 建议启用分页功能

### 文件存储配置

#### 配置项
- `file.base-path`: 文件存储基础路径
- `file.max-size`: 文件最大大小（字节）
- `file.allowed-types`: 允许的文件类型列表
- `file.enable-md5-check`: 是否启用MD5校验
- `file.temp-dir`: 临时文件目录
- `file.permanent-dir`: 永久文件目录
- `file.image-dir`: 图片文件目录
- `file.document-dir`: 文档文件目录
- `file.archive-dir`: 归档文件目录

#### 配置建议
1. 文件存储路径应使用绝对路径
2. 文件最大大小应根据实际需求设置
3. 允许的文件类型应根据业务需求配置
4. 建议启用MD5校验，确保文件完整性
5. 目录结构应清晰，便于管理

### Spring配置

#### 配置项
- `spring.application.name`: 应用名称
- `spring.servlet.multipart.max-file-size`: 单个文件最大大小
- `spring.servlet.multipart.max-request-size`: 请求最大大小

#### 配置建议
1. 应用名称应简洁明了
2. 文件大小限制应与文件存储配置一致
3. 请求大小限制应考虑实际业务需求

## 安全机制

### JWT认证

1. 用户登录后生成JWT令牌
2. 客户端在后续请求中携带JWT令牌
3. JWT拦截器验证令牌有效性
4. 从令牌中提取用户信息
5. 验证通过后放行请求

### 密码加密

1. 使用Argon2算法进行密码哈希
2. 密码哈希后存储在数据库中
3. 用户登录时验证密码哈希
4. 密码明文不存储在任何地方

### 文件类型验证

1. 上传文件时验证文件类型
2. 只允许配置的文件类型
3. 不允许的文件类型会被拒绝

### 文件大小限制

1. 上传文件时验证文件大小
2. 超过配置大小的文件会被拒绝
3. 防止大文件占用过多资源

## 系统初始化

### Root用户初始化

1. 系统启动时检查root用户是否存在
2. 如果不存在，创建root用户
3. 为root用户分配管理员角色
4. root用户拥有系统所有权限

## 配置最佳实践

### 安全配置

1. JWT密钥应使用强随机字符串，并定期更换
2. 数据库密码应使用强密码，不要使用默认密码
3. 生产环境应启用SSL连接
4. 文件存储路径应设置合适的权限

### 性能配置

1. 数据库连接池应根据实际负载配置
2. 文件大小限制应根据实际需求设置
3. 分页大小应根据数据量合理设置

### 可维护性配置

1. 配置文件应使用清晰的命名
2. 配置项应添加注释说明
3. 敏感信息应使用环境变量
4. 配置应支持多环境（开发、测试、生产）

## 注意事项

1. 生产环境的JWT密钥应与开发环境不同
2. 数据库密码不应明文存储在配置文件中
3. 文件存储路径应确保有足够的磁盘空间
4. 系统配置修改后需要重启应用才能生效
5. root用户初始化后，应修改默认密码
6. 文件类型和大小限制应根据实际业务需求调整
