# 生产管理模块

## 模块概述

生产管理模块是ERP系统的核心业务模块，负责管理生产计划、生产单位、工单、制剂配方等生产相关业务。该模块实现了从生产计划制定到生产执行的全流程管理，支持生产状态跟踪、生产数据统计等功能。

## 功能特性

- 生产计划管理：计划创建、编辑、删除、查询、状态管理
- 生产单位管理：生产单位信息维护
- 工单管理：工单创建、跟踪、结算
- 制剂管理：制剂信息维护、规格管理
- 制剂配方管理：配方组成、原料配比管理
- 生产状态跟踪：生产状态变更、状态日志记录
- 计划编号自动生成：支持自动生成唯一计划编号

## 核心接口

### 1. 生产计划管理接口

**控制器**: [ProductionPlanController.java](../src/main/java/com/tonghui/erp/Controller/ProductionPlanController.java)

#### 1.1 获取生产计划列表
- **路径**: `GET /api/production-plans`
- **参数**: 
  - `pageIndex`: 页码（从0开始）
  - `pageSize`: 每页数量
- **返回**: `PagedResult<ProductionPlan>`

#### 1.2 获取生产计划详情
- **路径**: `GET /api/production-plans/{id}`
- **参数**: `id` - 计划ID
- **返回**: `ProductionPlan`

#### 1.3 创建生产计划
- **路径**: `POST /api/production-plans`
- **参数**: `ProductionPlan`
- **返回**: `ProductionPlan`

#### 1.4 更新生产计划
- **路径**: `PUT /api/production-plans/{id}`
- **参数**: 
  - `id`: 计划ID
  - `ProductionPlan`: 计划信息
- **返回**: `ProductionPlan`

#### 1.5 删除生产计划
- **路径**: `DELETE /api/production-plans/{id}`
- **参数**: `id` - 计划ID
- **返回**: `boolean`

#### 1.6 高级查询生产计划
- **路径**: `GET /api/production-plans/search`
- **参数**:
  - `planNumber`: 计划编号（模糊匹配）
  - `relatedOrder`: 关联单号（模糊匹配）
  - `preparationCode`: 制剂编码（模糊匹配）
  - `preparationName`: 制剂名称（模糊匹配）
  - `currentStatus`: 当前状态（精确匹配）
  - `isArchived`: 是否归档（精确匹配）
  - `createdTimeStart`: 创建时间起始
  - `createdTimeEnd`: 创建时间结束
  - `updatedTimeStart`: 更新时间起始
  - `updatedTimeEnd`: 更新时间结束
  - `pageIndex`: 页码
  - `pageSize`: 每页数量
- **返回**: `PagedResult<ProductionPlan>`

#### 1.7 生成计划编号
- **路径**: `GET /api/production-plans/generate-plan-number`
- **返回**: 生成的计划编号

**编号格式**: `Plan + 年月日 + 4位序列号`  
**示例**: `Plan202602050001`

#### 1.8 更改生产计划状态
- **路径**: `POST /api/production-plans/{planId}/status/change`
- **参数**:
  - `planId`: 计划ID
  - `newStatus`: 新状态
  - `operatorId`: 操作员ID
  - `remark`: 备注
  - `finishedQuantity`: 成品数量（可选）
  - `productionCycle`: 生产周期（可选）
  - `yieldRate`: 得率（可选）
  - `unitPrice`: 单价（可选）
- **返回**: `boolean`

#### 1.9 恢复生产计划状态
- **路径**: `POST /api/production-plans/{planId}/status/resume`
- **参数**:
  - `planId`: 计划ID
  - `operatorId`: 操作员ID
  - `remark`: 备注
- **返回**: `boolean`

## 业务流程

### 生产计划创建流程

1. 系统自动生成计划编号
2. 用户填写计划基本信息
3. 选择制剂和规格
4. 设置计划数量和生产单位
5. 提交创建
6. 计划状态初始化为"待审批"

### 生产状态变更流程

1. 用户发起状态变更请求
2. 系统记录变更前状态
3. 更新计划状态
4. 记录状态变更日志
5. 记录操作人和操作时间
6. 记录变更原因

### 制剂配方管理流程

1. 创建制剂基本信息
2. 添加配方原料
3. 设置各原料的处方量
4. 保存配方信息
5. 配方可用于生产计划

## 生产状态

系统支持以下生产状态：

- **待审批**: 计划创建后的初始状态
- **已审批**: 计划已通过审批
- **生产中**: 计划正在执行生产
- **已完成**: 生产已完成
- **已暂停**: 生产暂停
- **已取消**: 计划被取消
- **已归档**: 计划已归档

## 业务规则

### 计划编号规则

1. 格式：`Plan + 年月日 + 4位序列号`
2. 每天序列号从0001开始
3. 系统自动生成，保证唯一性

### 状态变更规则

1. 状态变更需要记录操作人和时间
2. 状态变更需要记录变更原因
3. 某些状态之间有严格的流转顺序
4. 已归档的计划不能再次变更状态

### 制剂配方规则

1. 一个制剂可以有多个配方明细
2. 每个配方明细对应一种原料
3. 处方量必须大于0
4. 单位必须与原料单位一致

## 使用示例

### 创建生产计划

```json
POST /api/production-plans
{
  "planNumber": "Plan202602050001",
  "relatedOrder": "SO20260205001",
  "preparationCode": "ZJ-001",
  "preparationName": "葡萄糖注射液",
  "planQuantity": 1000,
  "productionUnit": "第一生产车间",
  "unitName": "瓶",
  "remark": "紧急订单"
}
```

### 更改生产计划状态

```http
POST /api/production-plans/1/status/change?newStatus=生产中&operatorId=1&remark=开始生产
```

### 查询生产计划

```http
GET /api/production-plans/search?pageIndex=0&pageSize=20&currentStatus=生产中
```

### 生成计划编号

```http
GET /api/production-plans/generate-plan-number
```
