### 用户权限模块设计问题

| 问题 | 严重度 | 说明 |
|------|--------|------|
| 缺少软删除字段 | 中 | 所有表无 `is_deleted`/`deleted_at` 字段，删除操作为物理删除，无法恢复 |
| 缺少登录日志表 | 高 | 无登录记录审计，无法追踪异常登录、登录IP、登录设备 |
| 缺少密码修改记录 | 中 | 无法追踪密码修改历史，密码策略难以审计 |
| department缺少状态字段 | 低 | 部门表仅2个字段，无状态、无层级关系（parent_id）、无排序字段 |
| department缺少父级ID | 中 | 无法表示部门层级关系（如：公司→部门→小组） |
| user表gender类型不当 | 低 | 使用VARCHAR(50)存储性别，应为TINYINT或ENUM |
| user缺少email字段 | 低 | 用户表缺少邮箱字段，可能影响通知功能 |
| 缺少操作日志表 | 高 | 无全局操作审计日志，关键操作无法追溯 |

### 物料基础数据模块设计问题

| 问题 | 严重度 | 说明 |
|------|--------|------|
| unit和dosage_form缺少状态字段 | 中 | 无法禁用/启用某个单位或剂型，数据只能物理删除 |
| unit和dosage_form缺少创建/更新时间 | 中 | 无法审计数据变更历史 |
| material.category_name用字符串存储 | 中 | 分类用VARCHAR存储而非外键关联分类表，数据冗余且不一致 |
| material.unit_name用字符串存储 | 中 | 单位用VARCHAR存储而非外键关联unit表，同上问题 |
| 缺少物料分类表 | 高 | 无独立的物料分类管理表，分类只能通过字符串硬编码 |
| material缺少规格单位字段 | 低 | 规格字段spec与unit_name职责不清晰 |
| symbol字段VARCHAR(255)过长 | 低 | 单位符号通常很短，VARCHAR(20)更合理 |

### 文件管理模块设计问题

| 问题 | 严重度 | 说明 |
|------|--------|------|
| 缺少文件版本控制 | 中 | 无法追踪文件修改历史，每次更新只能覆盖 |
| 缺少文件访问权限控制 | 中 | 无权限字段，文件访问权限只能通过业务层控制 |
| business_type用字符串存储 | 中 | 多态关联无法添加外键约束 |
| 缺少软删除字段 | 低 | 删除为物理删除，无法恢复 |
| 缺少下载次数/访问次数统计 | 低 | 无法追踪文件使用情况 |
| file_path与file_url职责重叠 | 低 | 两个路径字段定义不清晰，建议统一为一个 |

## 八、全局问题汇总

### 8.1 架构层面问题

| 问题 | 严重度 | 说明 |
|------|--------|------|
| 无软删除机制 | 高 | 全局缺少is_deleted/deleted_at字段，所有删除操作为物理删除 |
| 无操作审计日志表 | 高 | 关键业务操作（增删改）无统一审计日志 |
| 主键类型不统一 | 高 | INT/BIGINT混用，部分表无主键（stock_in_sequence/stock_out_sequence） |
| 状态字段类型不统一 | 中 | INT/VARCHAR(50)/VARCHAR(255)混用，应统一为VARCHAR(50)或ENUM |
| 无乐观锁字段 | 中 | 无version字段，高并发场景下可能出现数据覆盖 |

### 8.2 外键关系问题

| 问题 | 严重度 | 说明 |
|------|--------|------|
| 大量字符串替代外键 | 高 | material.category_name/material.unit_name/preparation.dosage_form等应为外键 |
| 多态关联无法约束 | 中 | stock.item_type+item_id/stock_transaction.related_type+related_id等无法添加外键 |
| 部分外键缺失 | 中 | equipment.room_id/stock_in.supplier_id等逻辑外键未声明 |

### 8.3 命名规范问题

| 问题 | 严重度 | 说明 |
|------|--------|------|
| 时间字段命名不一致 | 中 | created_time/created_at/create_time混用 |
| 状态字段命名不一致 | 中 | status/user_status/role_status/perm_status等前缀不统一 |
| 关联表主键命名不一致 | 低 | userrole_id/roleperm_id/userdepartment_id格式不统一 |
