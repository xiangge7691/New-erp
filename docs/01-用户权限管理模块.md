# 用户权限管理模块

## 模块概述

用户权限管理模块是ERP系统的核心基础模块，负责管理系统的用户、角色、权限和部门信息。该模块实现了基于角色的访问控制（RBAC）模型，支持用户认证、授权和权限管理。

## 功能特性

- 用户管理：用户创建、编辑、删除、查询
- 角色管理：角色定义、角色分配、角色权限绑定
- 权限管理：权限树构建、权限分配、权限查询
- 部门管理：部门结构管理、用户部门关联
- 认证授权：JWT令牌认证、密码加密（Argon2）

## 核心接口

### 1. 用户管理接口

**控制器**: [UserController.java](../src/main/java/com/tonghui/erp/Controller/UserController.java)

#### 1.1 获取用户列表
- **路径**: `GET /api/User`
- **参数**: 
  - `pageIndex`: 页码（从0开始）
  - `pageSize`: 每页数量
- **返回**: `PagedResult<UserDto>`

#### 1.2 获取用户详情
- **路径**: `GET /api/User/{id}`
- **参数**: `id` - 用户ID
- **返回**: `UserDto`

#### 1.3 创建用户
- **路径**: `POST /api/User`
- **参数**: `UserCreateDto`
  - `userName`: 用户名
  - `name`: 真实姓名
  - `password`: 密码
  - `phone`: 联系电话
  - `gender`: 性别
  - `status`: 状态
  - `notes`: 备注
  - `roleIds`: 角色ID列表
  - `departmentIds`: 部门ID列表
- **返回**: `UserDto`

#### 1.4 更新用户
- **路径**: `PUT /api/User/{id}`
- **参数**: 
  - `id`: 用户ID
  - `UserCreateDto`: 用户信息
- **返回**: `UserDto`

#### 1.5 删除用户
- **路径**: `DELETE /api/User/{id}`
- **参数**: `id` - 用户ID
- **返回**: `boolean`

#### 1.6 搜索用户
- **路径**: `GET /api/User/search`
- **参数**:
  - `userName`: 用户名关键词（模糊查询）
  - `departmentId`: 部门ID
  - `roleId`: 角色ID
  - `status`: 用户状态
  - `pageIndex`: 页码
  - `pageSize`: 每页数量
- **返回**: `PagedResult<UserDto>`

### 2. 认证接口

**控制器**: [AuthController.java](../src/main/java/com/tonghui/erp/Controller/AuthController.java)

#### 2.1 用户登录
- **路径**: `POST /api/auth/login`
- **参数**: `LoginRequest`
  - `userName`: 用户名
  - `password`: 密码
- **返回**: `AuthLoginResponse`
  - `token`: JWT访问令牌
  - `refreshToken`: 刷新令牌

### 3. 角色管理接口

**控制器**: [RoleController.java](../src/main/java/com/tonghui/erp/Controller/RoleController.java)

#### 3.1 获取角色列表
- **路径**: `GET /api/Role`
- **参数**: 分页参数
- **返回**: `PagedResult<RoleDto>`

#### 3.2 创建角色
- **路径**: `POST /api/Role`
- **参数**: `RoleCreateDto`
- **返回**: `RoleDto`

#### 3.3 更新角色
- **路径**: `PUT /api/Role/{id}`
- **参数**: 角色ID和更新数据
- **返回**: `RoleDto`

#### 3.4 删除角色
- **路径**: `DELETE /api/Role/{id}`
- **参数**: 角色ID
- **返回**: `boolean`

#### 3.5 为角色分配权限
- **路径**: `POST /api/Role/{roleId}/permissions`
- **参数**: 
  - `roleId`: 角色ID
  - `permissionIds`: 权限ID列表
- **返回**: `boolean`

### 4. 权限管理接口

## 业务流程

### 用户登录流程

1. 用户提交用户名和密码
2. 系统验证用户名和密码
3. 验证成功后生成JWT令牌
4. 返回令牌给客户端
5. 客户端在后续请求中携带令牌
6. 系统通过JWT拦截器验证令牌有效性

### 用户创建流程

1. 检查用户名是否已存在
2. 使用Argon2算法加密密码
3. 创建用户记录
4. 分配角色（如果提供）
5. 分配部门（如果提供）
6. 返回完整的用户信息

### 权限验证流程

1. 用户登录获取JWT令牌
2. 每次API请求携带JWT令牌
3. JWT拦截器验证令牌
4. 从令牌中提取用户信息
5. 根据用户角色和权限进行访问控制

## 数据传输对象（DTO）

### UserDto
用户信息传输对象，包含用户基本信息、角色列表、部门列表。

### RoleDto
角色信息传输对象，包含角色基本信息和权限列表。

### PermissionDto
权限信息传输对象，支持树形结构。

### DepartmentDto
部门信息传输对象。

### LoginRequest
登录请求对象，包含用户名和密码。

### AuthLoginResponse
登录响应对象，包含访问令牌和刷新令牌。

## 安全机制

### 密码加密
使用Argon2算法进行密码哈希，提供高强度的密码保护。

### JWT认证
- 令牌有效期：120分钟
- 刷新令牌有效期：10080分钟（7天）
- 令牌签发者：ErpSys
- 令牌受众：ErpSysUsers

### 权限控制
- 基于角色的访问控制（RBAC）
- 支持权限树结构
- 支持多角色分配
- 支持主部门概念
