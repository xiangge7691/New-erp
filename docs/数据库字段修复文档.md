## 三、执行步骤

### 3.1 备份数据库

```bash
mysqldump -u root -p erp_db > erp_db_backup_$(date +%Y%m%d).sql
```

### 3.2 执行修复脚本

```bash
mysql -u root -p erp_db < db/fix_database.sql
```

### 3.3 验证修复结果

```bash
# 检查表数量
mysql -u root -p -e "SELECT COUNT(*) AS table_count FROM information_schema.tables WHERE table_schema = 'erp_db';"

# 检查字段命名一致性
mysql -u root -p -e "SELECT table_name, column_name FROM information_schema.columns WHERE table_schema = 'erp_db' AND column_name IN ('created_time', 'updated_time', 'create_time', 'update_time');"
```

### 3.4 重启应用

```bash
./mvnw spring-boot:run
```

## 四、注意事项

1. **数据迁移**：如果现有数据使用了旧字段名（如`category_name`），需要编写数据迁移脚本将数据填充到新字段（如`category_id`）
2. **外键约束**：修复脚本会添加外键约束，如果存在脏数据可能报错，建议先清理数据
3. **字段冲突**：如果某些字段已存在，ALTER语句会报错但不影响其他语句执行
4. **备份优先**：执行前务必备份数据库
