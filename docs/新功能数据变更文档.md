# ERP系统新功能数据变更文档

> 本文档定义新功能所需的数据表结构和API接口设计。
> 变更范围：仅新增表+最小ALTER，不修改现有表结构。

---

## 一、变更概览

| 功能模块 | 变更类型 | 内容 |
|----------|----------|------|
| 岗位信息 | 新增表 | position |
| 人员档案 | 新增表 | personnel_file |
| 设备维保 | 新增表 | equipment_maintenance |
| 制剂工序模版 | 新增表 | preparation_process_template |
| 制剂文档 | 新增表 | preparation_document |
| 生产详情 | ALTER TABLE | production_process_record +4时间字段 |
| 首页数据 | 无表变更 | 复用现有表，新增Dashboard API |

---

```sql
SELECT * FROM personnel_file 
WHERE status = 1 
  AND health_cert_expire IS NOT NULL 
  AND health_cert_expire BETWEEN CURDATE() AND DATE_ADD(CURDATE(), INTERVAL ? DAY)
ORDER BY health_cert_expire ASC
```

**索引**：`idx_equipment_id(equipment_id)`, `idx_next_maintenance(next_maintenance_date)`

**API接口**：
```
GET    /api/equipmentMaintenance              分页查询
GET    /api/equipmentMaintenance/{id}         根据ID查询
POST   /api/equipmentMaintenance              新增
PUT    /api/equipmentMaintenance/{id}         修改
DELETE /api/equipmentMaintenance/{id}         删除
GET    /api/equipmentMaintenance/reminder     维保提醒（参数：days=7）
```

**维保提醒查询逻辑**：
```sql
SELECT em.*, e.equipment_name FROM equipment_maintenance em
JOIN equipment e ON em.equipment_id = e.equipment_id
WHERE em.next_maintenance_date IS NOT NULL
  AND em.next_maintenance_date BETWEEN CURDATE() AND DATE_ADD(CURDATE(), INTERVAL ? DAY)
ORDER BY em.next_maintenance_date ASC
```

## 三、ALTER TABLE变更

### 3.1 production_process_record - 新增时间字段

```sql
ALTER TABLE `production_process_record`
    ADD COLUMN `production_start` DATETIME DEFAULT NULL COMMENT '生产开始时间' AFTER `end_time`,
    ADD COLUMN `production_end`   DATETIME DEFAULT NULL COMMENT '生产结束时间' AFTER `production_start`,
    ADD COLUMN `inspection_start` DATETIME DEFAULT NULL COMMENT '检验开始时间' AFTER `production_end`,
    ADD COLUMN `inspection_end`   DATETIME DEFAULT NULL COMMENT '检验结束时间' AFTER `inspection_start`;
```

**用途**：订单状态总览页面显示生产详情时间线

## 四、首页Dashboard API

### 4.1 接口设计

```
GET /api/dashboard/summary
```

**返回数据**：
```json
{
  "code": 200,
  "data": {
    "productionStats": {
      "totalPlans": 100,
      "inProgress": 25,
      "completed": 60,
      "pending": 15
    },
    "stockWarnings": {
      "lowStock": 8,
      "expiringSoon": 5,
      "expired": 2
    },
    "approvalStats": {
      "pendingApproval": 12,
      "myPending": 3
    },
    "recentActivities": [...]
  }
}
```

**数据来源**：
- 生产计划统计：`production_plan` 表按 `current_status` 分组
- 库存预警：`stock` 表查询 `quantity <= min_quantity` 或 `expiry_date <= DATE_ADD(NOW(), INTERVAL 30 DAY)`
- 审批统计：`approval_instance` 表按 `status` 分组
- 最近活动：`operation_log` 表按 `created_at` 倒序
