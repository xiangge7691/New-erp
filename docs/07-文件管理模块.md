# 文件管理模块

## 模块概述

文件管理模块负责管理系统中各类文件的存储、上传、下载和删除操作。该模块支持文件分类管理、业务关联、文件类型验证、文件大小限制、MD5校验等功能，确保文件的安全性和可追溯性。

## 功能特性

- 文件上传：支持多种文件类型的上传
- 文件下载：提供文件下载功能
- 文件删除：安全删除文件
- 文件查询：支持按分类、业务信息查询文件
- 文件分类：支持文件分类管理
- 业务关联：支持文件与业务单据关联
- 文件类型验证：验证上传文件的类型
- 文件大小限制：限制上传文件的大小
- MD5校验：计算文件MD5值，用于文件完整性校验
- Base64编码：支持文件Base64编码

## 核心接口

### 文件管理接口

**控制器**: [FileController.java](../src/main/java/com/tonghui/erp/Controller/FileController.java)

#### 1.1 上传文件
- **路径**: `POST /api/files/upload`
- **参数**:
  - `file`: 文件对象（MultipartFile）
  - `category`: 文件分类（可选）
  - `description`: 文件描述（可选）
  - `businessId`: 业务ID（可选）
  - `businessType`: 业务类型（可选）
- **返回**: `FileInfo`

#### 1.2 下载文件
- **路径**: `GET /api/files/{id}`
- **参数**: `id` - 文件ID
- **返回**: 文件流

#### 1.3 删除文件
- **路径**: `DELETE /api/files/{id}`
- **参数**: `id` - 文件ID
- **返回**: `boolean`

#### 1.4 获取文件信息
- **路径**: `GET /api/files/info/{id}`
- **参数**: `id` - 文件ID
- **返回**: `FileInfo`

#### 1.5 分页查询文件列表
- **路径**: `GET /api/files/list`
- **参数**:
  - `category`: 文件分类（可选）
  - `pageIndex`: 页码（从0开始）
  - `pageSize`: 每页数量
- **返回**: `PagedResult<FileInfo>`

#### 1.6 根据业务信息查询文件
- **路径**: `GET /api/files/business`
- **参数**:
  - `businessId`: 业务ID
  - `businessType`: 业务类型
- **返回**: `List<FileInfo>`

## 业务流程

### 文件上传流程

1. 用户选择文件
2. 系统验证文件类型
3. 系统验证文件大小
4. 计算文件MD5值
5. 生成存储文件名
6. 保存文件到磁盘
7. 保存文件信息到数据库
8. 返回文件信息

### 文件下载流程

1. 用户请求下载文件
2. 系统查询文件信息
3. 系统读取文件
4. 返回文件流

### 文件删除流程

1. 用户请求删除文件
2. 系统查询文件信息
3. 删除磁盘上的文件
4. 删除数据库中的文件记录
5. 返回删除结果

### FileStorageService

**接口**: [FileStorageService.java](../src/main/java/com/tonghui/erp/Service/FileStorageService.java)  
**实现**: [FileStorageServiceImpl.java](../src/main/java/com/tonghui/erp/Service/impl/FileStorageServiceImpl.java)

**核心方法**:
- `encodeFileToBase64(MultipartFile file)`: 将文件转换为Base64字符串
- `calculateMD5(MultipartFile file)`: 计算文件的MD5值
- `uploadFile(MultipartFile file, String category, String description)`: 上传文件到磁盘并保存元数据
- `uploadFileWithBusiness(MultipartFile file, String category, String description, Long businessId, String businessType)`: 上传文件到磁盘并保存元数据（带业务信息）
- `getFileInputStream(Long fileId)`: 根据文件ID获取文件输入流
- `deleteFile(Long fileId)`: 删除文件
- `isAllowedFileType(MultipartFile file)`: 验证文件类型是否允许
- `isFileSizeExceeded(MultipartFile file)`: 验证文件大小是否超出限制
- `updateBusinessInfo(FileInfo fileInfo, Long businessId, String businessType)`: 更新文件的业务关联信息

## 文件类型

系统支持以下文件类型：

### 图片类型
- `image/jpeg`: JPEG图片
- `image/png`: PNG图片
- `image/gif`: GIF图片

### 文档类型
- `application/pdf`: PDF文档
- `application/msword`: Word文档（.doc）
- `application/vnd.openxmlformats-officedocument.wordprocessingml.document`: Word文档（.docx）
- `application/vnd.ms-excel`: Excel文档（.xls）
- `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`: Excel文档（.xlsx）
- `text/plain`: 文本文件

## 文件分类

系统支持以下文件分类：

- `images`: 图片文件
- `documents`: 文档文件
- `archives`: 归档文件
- `temp`: 临时文件

## 存储类型

- `LOCAL`: 本地存储
- `CLOUD`: 云存储（预留）

## 业务类型

系统支持以下业务类型：

- `PURCHASE_ORDER`: 采购订单
- `PRODUCTION_PLAN`: 生产计划
- `STOCK_IN`: 入库单
- `STOCK_OUT`: 出库单
- `APPROVAL`: 审批单据

## 业务规则

### 文件上传规则

1. 文件大小不能超过10MB
2. 文件类型必须在允许的类型列表中
3. 文件名不能包含特殊字符
4. 文件上传后自动生成MD5值

### 文件存储规则

1. 文件按分类存储在不同目录
2. 存储文件名使用UUID生成，避免冲突
3. 原始文件名保存在数据库中
4. 文件路径保存在数据库中

### 文件删除规则

1. 删除文件前检查文件是否存在
2. 删除磁盘上的文件
3. 删除数据库中的文件记录
4. 删除操作不可恢复

### 文件查询规则

1. 支持按分类查询文件
2. 支持按业务信息查询文件
3. 支持分页查询
4. 查询结果按创建时间倒序排列

```yaml
  allowed-types:
    - image/jpeg
    - image/png
    - image/gif
    - application/pdf
    - application/msword
    - application/vnd.openxmlformats-officedocument.wordprocessingml.document
    - application/vnd.ms-excel
    - text/plain
  enable-md5-check: true
  temp-dir: temp
  permanent-dir: permanent
  image-dir: images
  document-dir: documents
  archive-dir: archives
```

## 使用示例

### 上传文件

```http
POST /api/files/upload
Content-Type: multipart/form-data

file: [文件]
category: documents
description: 采购订单附件
businessId: 100
businessType: PURCHASE_ORDER
```

### 下载文件

```http
GET /api/files/1
```

### 删除文件

```http
DELETE /api/files/1
```

### 查询文件列表

```http
GET /api/files/list?category=documents&pageIndex=0&pageSize=20
```

### 根据业务信息查询文件

```http
GET /api/files/business?businessId=100&businessType=PURCHASE_ORDER
```


## 注意事项

1. 文件上传前应验证文件类型和大小
2. 文件删除后无法恢复，请谨慎操作
3. 文件MD5值用于文件完整性校验，不应修改
4. 文件路径不应直接暴露给用户，应通过接口访问
5. 文件上传可能占用较多带宽，建议使用分片上传
6. 文件存储路径应定期备份，防止数据丢失
